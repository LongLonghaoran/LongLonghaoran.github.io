PHP部分:
    1.介绍服务端开发
        除了 html js css以外，还需要 web服务器 http 服务端开发 数据库技术 ajax
    2.搭建web服务器
        1.下载一个比较常用的web服务器软件 Apache HTTP server monitor
            按照老师推荐，是下载的绿色版的
        2.安装包内有 httpd这个工具，借助它来安装 apache server
            httpd.exe -k install -n '自定义一个服务名称'
        3.安装完毕后会提示，有Syntax 错误,这是 安装目录/config/httpd.conf
            文件中有配置ServerRoot 而默认是 c:/apache24这类的路径，改成我们实际
            的安装路径即可
        4.安装完毕后可以继续使用命令行 httpd -k start或者在monitor图形化界面中点击start
        都可以启动
        5.默认情况下，apache服务会监听htdocs目录，可以将静态文件放在这个目录下
            我们可以更改，比如改到 E:/www,刚改完的时候，浏览器中访问它会提示forbiden 没权限
        6.总之，不管是服务根目录 还是 监听端口这些，都是在 apache安装目录/conf/httpd.conf文件中配置
        7.当更改了httpd.conf配置文件之后，最后通过 httpd.exe -t来测试一下配置文件是否有误
            这是由于配置文件中,
                <Directory />
                    AllowOverride none
                    Require all denied
                </Directory>
            指定所有目录都被禁止访问了,并且配置文件也说明了，我们可以单独指定其他目录让
            它有权限访问，配置文件中有一段对于默认的根目录（即htdocs)的访问权限设置的，代码
            我们把它改一下即可：
                <Directory "E:/www">
                    #
                    # Possible values for the Options directive are "None", "All",
                    # or any combination of:
                    #   Indexes Includes FollowSymLinks SymLinksifOwnerMatch ExecCGI MultiViews
                    #
                    # Note that "MultiViews" must be named *explicitly* --- "Options All"
                    # doesn't give it to you.
                    #
                    # The Options directive is both complicated and important.  Please see
                    # http://httpd.apache.org/docs/2.4/mod/core.html#options
                    # for more information.
                    #
                    Options Indexes FollowSymLinks

                    #
                    # AllowOverride controls what directives may be placed in .htaccess files.
                    # It can be "All", "None", or any combination of the keywords:
                    #   AllowOverride FileInfo AuthConfig Limit
                    #
                    AllowOverride None

                    #
                    # Controls who can get stuff from this server.
                    #
                    Require all granted
                </Directory>
            改完再次重启服务,就可以访问了
        8.默认文档
            # DirectoryIndex: sets the file that Apache will serve if a directory
            # is requested.
            #
            <IfModule dir_module>
              DirectoryIndex index.html
            </IfModule>
            这里配置的是当我们访问的是一个目录，展示的默认文档，默认是index.html,
            当然，如果目录中没有index.html或者其他我们指定的文件夹的话，那么页面中
            会打印出目录的结构出来
        9.配置虚拟主机
            Apache可以像Nginx一样配置多台主机，将多个域名都转发到同一台主机的不同端口
            httpd.conf文件的Virtual Hosts配置项就可以配置多台主机，它默认是包含 conf/extra/httpd-vhosts.conf
            这个配置文件，解除这行注释，然后再去编辑这个配置文件
                <VirtualHost *:80>
                    ServerAdmin webmaster@dummy-host.example.com
                    DocumentRoot "E:/www"
                    ServerName dummy-host.example.com
                    ServerAlias www.dummy-host.example.com
                    ErrorLog "logs/dummy-host.example.com-error.log"
                    CustomLog "logs/dummy-host.example.com-access.log" common
                </VirtualHost>
            解析:
                VirtualHost *:80 表示监听本机任意ip地址的80端口的请求
                DocumentRoot "E:/www" 文档根目录
                ServerName 域名，浏览器可以通过这个域名直接访问,前提条件是访问这个域名的客户端，要能够通过dns以及这个域名来访问到你的主机，
                    在本地我们可以通过etc/hosts文件来实现，公网环境下需要购买域名
            按照这种方式那么我们就可以在一台主机上开多个站点, 需要注意的就是
                1.记得配置etc/hosts 映射新的域名
                2.记得配置新的目录权限(在httpd.conf中配置)
                    <Directory "E:/site1">
                        AllowOverride None
                        Require all granted
                    </Directory>
        10.第9步做完之后，浏览器访问域名的话，有可能仍然无法访问，因为浏览器会自动将请求转为https,
            这是因为某些域名后缀会导致服务器自动将其调转为https?(老师并没有解释)
        11.让Apache支持 PHP
            还是来到httpd.conf文件中，apache支持载入其他模块,使用LOAD_MODULE来加载模块
            PHP也有一个模块供apache加载,在php官网下载php，下载下来后有个apache_xx.dll的库，将这个文件的
            路径配置到httpd.conf中
                LoadModule php7_module E:\php-7.3.9-Win32-VC15-x64\php7apache2_4.dll

            然后还需要配置mime.types,在httpd.conf文件中利用 AddType来进行配置
            在IFModule模块中，添加AddType
                AddType application/x-httpd-php .php

    3.ip地址与防火墙
        连接至局域网时，网管会分配一个ip给我们
        如果不连接，那么本地也有一个回环地址 127.0.0.1
        可以禁止本地计算机的服务（比如apache）被外部请求到/访问到
    4.域名
        域名需要花钱注册的，ip地址的别名
    5.DNS
        通过宽带运营商的服务器解析一个域名背后对应的ip，这个过程叫做dns寻址，
        完成寻址过程的服务器叫做dns服务器
    6.hosts
        发起对dns服务器查询请求之前，会优先检查hosts文件
            mac: /etc/hosts
            windows: c/windows/system32/drivers/etc/hosts/
