3.封装一个ajax方法
    我们可以封装成这个样子
        function ajax(){
            var xhr= new XMLHttpRequest();
            xhr.open("get", "time.php");
            xhr.send(null);
            xhr.onreadystatechange = function(){
                if(this.readyState != 4) return;
                console.log("结果出来了!")
                return this.responseText;
            }
        }
      console.log(ajax());
4.关于ajax的返回值，注意这个函数的返回值
    上面的return是无效的！因为return this.responseText是处于onreadystatechange这个函数里面
    就是说，内部函数返回的值，无法通过外部函数的调用接收到

                console.log(this);
                console.log(res);
                console.log('成功了')
            }
        })
6.服务端指定 content-type
    服务端也可以返回数据格式为 application/json
    在$.ajax中，response会自动地根据响应的content-type来改变响应体的类型
    但是如果服务端返回时没有设置content-type的话，客户端获取的就会是json字符串了
    这个时候，有两种方式:
        1.我们手动 JSON.parse(res)
        2.jquery发送ajax请求时，可以设置dataType
            $.ajax({
                dataType:'json'
            })
            这样子jquery会自动的将json字符串转换为js对象
    一旦设置了dataType之后，就不再关心服务端响应的Content-Type了,也就是
        说，即便服务端返回时设置content-type为application/xml，客户端仍然会
        按照json去转换为js对象
7.Jquery ajax中的回调事件
    $.ajax({
        url:xxx,
        type: xx,
        success:function(res){
        },
        error:function(xhr){
        },
        complete: function(xhr){
        },
        beforeSend: function(xhr){
        }
    }
   之前在使用xhr对象的时候，在判断请求是否完成的时候，我们使用的是 readyState == 4
   其实这并不准确,readyState == 4，只是代表请求完成了，但是请求并不一定成功了
   比如 还有 402 403 404 500 502 504等错误可能出现，这个时候响应的结果是一个错误页面
   Jquery ajax中success就是表示请求完成而且成功了
   error表示请求处理失败了
   complete表示请求处理完成了
   此外还有beforeSend回调，在send发送请求之前执行
   另外还有一个注意点,只有success回调函数的参数是 响应体，其余的回调函数的参数
   得到的都是xhr对象

9.全局ajax事件
    如果我们有多个ajax请求需要执行，并且每次ajax的loading和结束阶段的显示效果
    都是一样的，那么我们会需要给每个ajax方法都来写上 loading效果的代码。
    我们可以给所有的ajax请求来绑定相同的回调函数：
        $(document).ajaxStart(function(){
            请求开始时需要执行的操作
        })
        $(document).ajaxEnd(function(){
            请求结束需要执行的操作
        })
10.配合 NProgress显示加载进度
    页面的刷新都采用 ajax来做，有一个专业的刷新的名词叫做 pjax
    然后在页面顶部做一个假的进度条,像github 等网站都是这么做的
    很简单，页面中引入nprogress.css 和 nprogress.js之后，写下如下两行即可
   <link rel="stylesheet" href="nprogress.css">
   <script src="nprogress.js"></script>
   <script>
        $(function($){
            $(document).ajaxStart(function(){
                NProgress.start();
            });
            $(document).ajaxStop(function(){
                NProgress.done();
            });
        });
    </script>
11.关于同源策略
    所谓同源，就是两个网站的 域名 协议 端口都一样，就表示网址是同源的,
    在默认情况下 两个不同源的地址之间是无法通过ajax进行交互的
    需要注意，即便不允许使用ajax进行，但是从客户端发送的请求还是能够到达
    服务端，并且服务端也会按照请求来返回响应，只不过，在客户端要读取响应的
    时候，被浏览器就禁止了。
    1.首先来总结一下可以发送请求的几种方式
        1.img标签，src属性
            比如:
                var img = new Image();
                img.src = 'http://ajax/time.php'
            设置了属性之后就会立即发送请求去获取
        2.link标签 href属性
            var link = document.createElement("link");
            link.rel = 'stylesheet'
            link.href = 'http://xxx'
            设置了rel再去设置href,在将其append到页面中之后就会发送请求进行预取
        3.background-image: url()
        4.iframe
        5.script标签  src属性
            同Link标签
                var script = document.createElement("script");
                script.src = "http://www.baidu.com"
                document.body.appendChild(script);
            另外如果请求的是一段js代码:
                比如<script src='alert.js' >
            或者响应结果的content-type是application/javascript
            那么浏览器不但会发送请求，而且在收到响应之后还会立即执行返回的
            js代码
    2.默认情况下浏览器发送请求时，会按照响应的content-type类型来决定如何处理响应:
        1.application/json
            按照Json处理
        2.application/javascript
            按照js来处理，收到响应后立即执行

    3.跨域的解决方案
        JSONP-------通过script请求一个服务端的php文件:
            这个文件返回的结果是一段Js,作用是调用我们事先定义好的一js个函数
            除此之外服务端也需要做配合，服务端需要返回一段Js代码，设置
            content-type为 application/javascript
            然后在返回的Js代码中调用这个js函数，并将生成的数据传入到函数中去即可
        例子:
            客户端:
                function jsonp (url, params, callback) {
                  var funcName = 'jsonp_' + Date.now() + Math.random().toString().substr(2, 5)
                  if (typeof params === 'object') {
                    var tempArr = []
                    for (var key in params) {
                      var value = params[key]
                      tempArr.push(key + '=' + value)
                    }
                    params = tempArr.join('&')
                  }
                  var script = document.createElement('script')
                  script.src = url + '?' + params + '&callback=' + funcName
                  document.body.appendChild(script)
                  window[funcName] = function (data) {
                    callback(data)
                    delete window[funcName]
                    document.body.removeChild(script)
                  }
                }

                jsonp('http://localhost/jsonp/server.php', { id: 123 }, function (res) {
                  console.log(res)
                })
            服务端:
                // 如果客户端采用的是 script 标记对我发送的请求
                // 一定要返回一段 JavaScript
                header('Content-Type: application/javascript');
                $result = json_encode($data);
                $callback_name = $_GET['callback'];
                echo "typeof {$callback_name} === 'function' && {$callback_name}({$result})";
        2.CORS跨域资源共享
           Cross Origin Resource Share 跨域资源共享
           其实，现在浏览器已经网站之间进行跨域访问了。只不过需要一些配置:
            只需要在服务端返回响应之前设置响应头
                Access-Control-Allow-Origin: *
            这样就会允许进行跨域请求访问
