2.DOM介绍
    1.
        元素: 页面中的所有的标签都是元素,元素可以看成是对象
        节点: 页面中所有的内容都是节点！注意：包括文本在内，文本也算是个 文本节点,
            页面中任何内容都能叫做节点
    2.
        文档： 一个页面就是一个文档
        层级关系就是:
            文档 > html > head + body> 其他标签
3.DOM操作:
    1.为元素注册一个事件:
        1.btn = document.getElementById('btn')可以获取一个元素
        2.通过这个元素可以设置它的任意属性,比如 btn.value btn.onclick
        3.绑定点击事件实质上就是设置它的onclick属性
            btn.onclick = function(){}
            需要注意，尽量使用onclick = 函数定义，直接将函数定义赋值给它
            而不是给一个函数名字，因为那样有可能因为函数而被覆盖
            function f(){}
            function f(){console.log("另一个");}
            btn.onclick = f; //错误，不建议这样
        4.因为获取的元素可以直接点它的属性，所以按照类似的方法还可以设置：
            document.getElementById("img").src = xx
            document.getElementById("btn").value = xx
            元素.innerText = "xxx"
            等等
        5.innerText但凡是被一个对元素包起来的文本都使用innerText来设置
    2.点击按钮设置所有p标签的文本
        document.getElementsByTagName("") 返回的是一个伪数组,意味着有length属性，并且可以[0] [1]通过索引来获取元素
        事实上，getElement方法的调用者是可以更改的，document是表示在整个
        文档中搜索：
            elements = element.getElementsByTagName(tagName)
            elements 搜索到的元素的动态HTML集合HTMLCollection，它们的顺序是在子树中出现的顺序。 如果没有搜索到元素则这个集合为空。
            element 搜索从element开始。请注意只有element的后代元素会被搜索，不包括元素自己。
            tagName 要查找的限定名。 字符 "*" 代表所有元素。 考虑到兼容XHTML，应该使用小写。
    3.例子：改变多个input的值,但是不要改变input type='button'的值:
        document.getElementById("btn").onclick = function(){
            var pObjs = document.getElementsByTagName("input");
            for(var i =0;i<pObjs.length;i++){
                if(pObjs[i].type=="button"){//可以直接.type获取type的值，而且这个值就是个字符串

                }else{
                    pObjs[i].value="我要改它的值";
                }
            }
        }
    4.再次说明，只要在标签中出现的属性，都可以作为对象属性点出来访问并修改
        比如: 下拉框中可以设置 selected=true
              复选框 checked=true
              单选框 checked=true
              文本框 disabled=true
    5.在某个元素的事件中，自己的事件中的this表示当前的元素对象
        var objs = document.getElementsByTagName("img");
        for(var i=0;i<objs.length;i++){
            this.onclick = function(){
                alert('被点击了');
            }
        }
    6.一个注意点：循环给多个元素注册事件时，请不要在事件函数中使用循环变量:
        比如:
            var objs = document.getElementsByTagName("img");
            for(var i=0;i<objs.length;i++){
                objs[i].onclick = function(){
                    console.log(objs[i].src); //错误！事件函数是在事件触发的时候才执行，这里i=objs.length+1
                }
            }
        由于事件函数是在事件触发的时候才会执行，所以当真正执行事件函数的时候，其实循环
        已经执行完了，而且i值已经等objs.length+1了，所以objs[i]是undefined，会报错
    7.超链接的事件函数中，如果最后return false,就不会跳转了
    8.设置表单标签的一个规律:
        在表单标签中，如果属性和值，如果属性和值只有一个，并且值是这个属性本身，那么，在dom
        操作的时候，这个属性值，是boolean类型就可以了
    9.有了上面的基础，尤其是”只要在标签中出现的属性，都可以作为对象属性点出来访问并修改"
        这一句，我们可以知道，设置样式其实也可以，通过style属性即可,
        style自身也是个对象,所以可以通过style属性设置样式
            比如:
                div.style.width = '300px'
        需要注意的是:
             单反是css中多个单词的写法，那么在js中命名规则均改为驼峰式写法即可
             比如:
                div.style.backgroundColor = 'red'
    10.通过类样式来实现改变样式
        还是按照 xx.属性名 = 'xx'
        还可以设置元素的类名: element.className = "类名"

    11.阻止超链接跳转:
        有几种方式可选择，原理各不相同:
            1.给它一个onclick事件:事件函数执行return false;
                函数碰到return false就会停止执行,超链接跳转其实也处于单击事件
                执行的过程之中，而return false即表示整个单击事件结束,不会再做
                其他操作了，所以就不会跳转了，或者换句话说，由于a标签的事件函数
                的返回值被a标签给接收了，而a标签一看到是false所以就不执行后续
                操作了
                一个注意点，一个小坑:
                    如果给a标签的onclick是一个命名函数，那么一定要在命名函数的
                    前面加上return 才可以生效
                    <a href='http://www.cc.com' onclick='return f()'>
                    <script>
                        function f(){
                            return false;
                        }
                    </script>
                    因为onclick里面就已经是个函数了，再在这里调用函数的话，只能相当于
                    是函数里面调用函数，所以必须再一次地将f()的返回值进行返回才行
            2.使用事件注册,这样也可以做到阻止超链接的跳转
                document.getElementById("a1").onclick = function(){
                    return false;
                }

4.包装innerText和textContent
    其实innerText是ie8的标准属性,谷歌浏览器纯粹是由于它自身兼容性好所以才支持
    这个属性的,但是像火狐较低版本的浏览器就不支持了,
    类似的 textContent 在ie8不支持
    面对这种情况，我们需要写兼容代码:
        解决方案，当浏览器不支持一个属性时，它的返回值将会是undefined:
          //兼容代码

          //设置任意的标签中间的任意文本内容
          function setInnerText(element,text) {
            //判断浏览器是否支持这个属性
            if(typeof element.textContent =="undefined"){//不支持
              element.innerText=text;
            }else{//支持这个属性
              element.textContent=text;
            }
          }

          //获取任意标签中间的文本内容
          function getInnerText(element) {
            if(typeof element.textContent=="undefined"){
             return element.innerText;
            }else{
              return element.textContent;
            }
          }
5.innerText 和 innerHTML的区别:
    通过innerText只会设置文本内容，即便你编写了html内容在里面，js引擎也会将其
    自动转义，比如将>转移为 &gt;,同样获取内容时，元素标签将被忽略
    而innerHTML是有标签效果的，获取内容时也会将标签一同展示，设置内容时，可以
    设置为html标签
6.自定义属性
    这个我已经用过无数次了..当然，还是有一点需要补充的：
    自定义的属性无法通过dom对象直接点出来，比如我给元素加了个score属性
    <span score=90>龙浩然</span>
    document.querySelector("span").score //错误
    1.dom对象虽然可以点出元素的属性，但是它并非直接去实时的读取目前的属性，
    实际上是在dom树种维护了一套元素的属性，它里面拥有的默认属性才可以被
    点出来，我们自定义的属性只能通过getAttribute("")来获取,通过setAttrbute来设置,
    只有这样才可以将属性实实在在的设置到dom实体对象上面
    2.移除元素属性，
        不论是自定义属性还是元素自带属性，只要是个属性，都是通过removeAttrbute移除
        element.removeAttribute("属性名")
