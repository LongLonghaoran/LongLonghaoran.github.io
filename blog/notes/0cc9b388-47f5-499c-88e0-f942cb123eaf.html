1.关于模块化
1.Javascript天生不支持模块化,require exports这些方法、概念是node中才引入的

2.浏览器要做到模块化，目前也有很多其他方式
    1.require.js AMD
    2.sea.js     CMD
    3.es 6 modules官方规范
    4.UMD

3.在2015年发布的ES6标准，已经宣布js支持模块化，之后在这之后的几年还有很多浏览器不支持



2.关于package-lock文件
    
npm init -y 生成一个Package.json文件
npm --save 文件名 还会生成一个 package-lock.json文件

在npm install 的时候，在安装完需要的包之后，会继续查找该包下的package.json文件，进一步去安装该包依赖的
其他包，依次重复，直至把所有的包都安装完成，但是这个过程是需要很多时间的，于是就有了package-lock.json这个
文件，它会保存node_modules中的包依赖信息

3.npm 5.xx之后的版本，在安装包的时候，不用加 --save也会保存依赖信息

4.package-lock还可以锁定版本,在有package-lock的情况下，执行 npm install 会严格按照文件指定的版本进行安装
    如果没有lock文件的话，那么默认将会安装最新的版本的包,所以一般情况下只需将package-lock.json文件加入版本
    控制提交就可以了

7.NodeJs中使用mongo
    使用mongodb包,这是官方提供NodeJs连接mongodb的包,但是，使用它还是比较原生，比较麻烦的，实际使用
    中，常用的是 mongoose这个第三方包
1.安装 npm i mongoose

2.使用
const mongoose = require('mongoose');
//连接test数据库,这个数据库可以不用提前创建，如果不存在mongo会自动创建，但是是在插入第一条数据之后再创建
mongoose.connect('mongodb://localhost:27017/test', {useNewUrlParser: true, useUnifiedTopology: true});

//新建一条文档记录
//第一个参数传入一个大写的单数名词，mongoose会自动转为复数小写形式作为集合名称
const Cat = mongoose.model('Cat', { name: String });

//保存,异步保存，保存成功/失败后执行回调
const kitty = new Cat({ name: 'Zildjian' });
kitty.save(function(err){
  if(err){
      console.log(err);
  }  else {
      console.log('meow');
  }
})
3.使用mongoose可以以更加面向对象并类似关系型数据库的方式来使用数据
    1.比如可以定义表结构
        定义了表结构之后，如果在创建数据时，添加额外的字段的话，将会被忽略
    2.通过表结构创建数据
    
总的来讲:使用数据库的大致都可以分为以下几步:
    1.定义表结构，定义模型
    2.根据api使用模型
