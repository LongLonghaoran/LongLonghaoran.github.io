3.Vue中的全局过滤器
    允许自定义过滤器，被用于一些常见的文本格式化，常用在两个地方,mustache插值{{}} 和 v-bind
    定义过滤器:
        Vue.filter('过滤器名称', 回调函数(data, 传过来的其他参数){})
        定义过滤器需要在Vue实例化之前进行
    使用过滤器:
        msg | 过滤器名称(参数)  
        使用一个管道符连接
        注意上面调用的时候，是可以传递参数的
<div id="app">
    <p>{{  msg | msgFormat }}</p>
</div>
<script>
    //    自定义过滤器,Vue.filter(过滤器名称, 回调函数(之后将使用过滤器的数据))
    Vue.filter('msgFormat', msg => msg.replace('加油', '努力') );
    var vm = new Vue({
        el: '#app',
        data: {
            msg: '龙浩然，你一定要加油，只有靠自己才有出路!'
        },
        methods: {
        }
    })
</script>
    还可以通过管道符连接多个过滤器

4.私有过滤器
     通过Vue初始化选项的filters来配置:
        
var vm2 = new Vue({
    el: '#app2',
    data: {
        dt: new Date()
    },
    methods: {},
    filters: { //定义私有过滤器，有两个参数，过滤器名称，和处理函数
        timeFormat(datestr){
            var dt = new Date(datestr);
            return `${dt.getFullYear()}-${dt.getMonth()+1}-${dt.getDate()}-${dt.getHours()}:${dt.getMinutes()}:${dt.getSeconds()}asdasd`
        }
    }
})
调用方式和全局过滤器一样，当私有过滤器和全局过滤器名称一致，优先调用
私有过滤器

7.自定义指令让文本框获取焦点
//    定义全局指令, Vue.directive(指令的名称，一个对象)
    //调用的时候，名称前面需要加上v-来使用
    //第二个参数对象，中有一些函数，在特定条件会执行
    Vue.directive('focus', {
        inserted: (el) => {
        //在每个钩子函数中，第一个参数都是el，它是一个原生的JSDOM对象
            el.focus();
        }
    })
第二个参数对象中，提供了几种常用的钩子函数，他们会在特定条件下执行
    1.bind 绑定的时候，每当指令绑定到元素上的时候会立即执行这个bind函数，只执行一次
    需要注意，在元素刚刚绑定了指定的时候，还没有插入到DOM中去,这个钩子
    是在inserted之前执行的
    2.inserted 插入的时候
        元素插入到DOM中的时候，会执行inserted函数
    3.updated
        当组件VNode更新的时候会执行,可能会触发多次

钩子函数的参数:
    1.元素对象 JSDOM
    2.binding
        一个对象:
        name: 指令名
        value: 指令的绑定值
        arg: 传给指令的参数
        expression: 指令的绑定值的原始形式
    3.vnode
    4.oldVnode

私有指令, 通过Vue初始化参数中的directives属性配置
    new Vue({
        directives: {
            'fontWeight':{
                'bind': function(el, binding){}
            }
        }
    })

指令函数的简写形式
    如果需要同时绑定update 和 bing钩子，有一种简写形式
    全局形式:    
    Vue.directives('自定义指令名', function(el, binding){
    })
    私有形式：
    new Vue({
        direcitives: {
            "fontSize": function(el, binding){}
        }
    })

Vue的生命周期:
    生命周期钩子 == 生命周期函数 == 生命周期事件
    生命周期钩子通过Vue配置选项来配置:    
    1.beforeCreate
        Vue实例初始化之前就会执行
        在BeforeCreate生命周期函数执行的时候，data和methods都还没有被初始化

    2.created
        这个时候data methods等已经初始化完毕，可以调用了
    3.beforeMount
        这个时候，Vue已经初始化完毕并且模板已经在内存中编译完成了，
        但是还没有将模板渲染到页面中,注意： 这里只是Vue编译完成的内容
        还没渲染到页面，但是页面中是有原始的值的，比如 {{}} v-bind v-if
        这些原本就写在html中的内容还是会有的
    4.mounted
        将编译好的模板替换到页面中去,此时如果没有后续操作，那么vm Vue实
        例就留在内存当中
    组件运行阶段的声明周期函数
    5.beforeUpdated
        当Vue中的数据发生改变(比如 data中的数据发生改变)后但还没没有
        重新渲染页面时执行此钩子
        注意: 执行此函数时，数据已经改变，只不过页面还没有改变
    6.updated
        数据已经改变，且页面也重新渲染了之后执行此函数/钩子
    组件销毁阶段
    7.beforeDestroy
        已经从运行阶段进入到销毁阶段,所有的data和methods以及 filters过滤器
        direcitives指令，仍然都处于可用状态    
    8.Destroyed
        过滤器filters 指令direcitves 方法 methods 数据data都已经销毁，不可           访问了
