VUE简介:
    1.vue，前端三大框架之一（国内最火的框架）
    2.vue是一套构建用户界面的框架，只关注视图层，它不仅易于上手，还便于
    与第三方库或既有整合,vue只是负责MVC中的V(view)这一层面
    3.Node(后端)的MVC和前端的MVVM之间的区别:
        MVVM:
            分成了 M V VM层
            VM是是M和V之间的调度者
            M保存的是每个页面中单独的数据
            V就是页面中html结构

    2.v-cloak
        解决插值表达式的闪烁问题，在vue加载完成(显示是原始的插值的问
        题,在vue加载完成之后),在vue加载完成后,该属性会消失
    3.v-text 用于设置元素内容(类似于innerText)
        vue加载完成后将它的值v-text属性的值，赋予给元素作为innerText
    4.v-html 类似于innerHTML
        上面两个插入的字符串都只会解析成字符串来展示，即便书写的是html字
        符串,v-html则可以输出html
    5.v-bind 用于绑定属性 的一个指令
        如果我们想给任意一个属性来绑定值，那么可以使用v-bind:属性名="data中对应的key"
        其中的属性值，将会被当做js来执行，也就意味着还可以使用 data中的键 + 123  这样的表达式来表示了
    6.v-bind简写:
        v-bind可以直接省略，直接在元素属性名前面加上 : 冒号
    7.v-on 事件绑定
        直接通过 v-on:事件名= '要执行的事件处理代码'，例如: v-on:click='单击
        事件代码'
        注意:
            属性值，获取到变量、方法，会从methods中去查找，这意味window
            document alert这些对象都将获取不到
    8.v-on的缩写: @
        例如: @click='show'

跑马灯效果制作:
    es6中的函数，省略了 function关键字
        匿名函数的箭头函数形式: () => {}
        对象中的方法属性的匿名函数形式: 方法名(){} 
<!--创建一个控制区域-->
    <div id="app">
        <input type="button" value="浪起来" @click="lang">
        <input type="button" value="低调" @click="stop">
        <h4> {{msg}} </h4>
    </div>
    <script>
        var vm = new Vue({
            el: '#app',
            data: {
                msg: '猥琐发育，别浪~~!',
                timeId: null
            },
            methods: {
                lang(){
                    if(this.timeId === null){
                        _this = this;
                        var timeId = setInterval(() => {
                                    console.log(vm);
//                    在VM内部如果想要data对象中的属性，或者methods方法，都要使用this,this表示是当前VUE实例
                        var start = _this.msg.substring(0, 1);
                        var end = _this.msg.substring(1);
//                    由于双向绑定是动态的，只要更改data中的属性值，页面上对应的绑定值会自动更改
                        _this.msg = end + start;
                    }, 400);
                        this.timeId = timeId;
                    }
                },
                stop(){
                    clearInterval(this.timeId);
                    this.timeId = null;
                }
            }
        })
    </script>
事件修饰符介绍:
    stop 阻止事件冒泡
    prevent 阻止事件默认行为，比如a链接的跳转，表单的提交
    capture 添加事件侦听器时使用事件捕获模式(从外往里触发)
    self 只有自身才会触发事件
    once 只触发一次事件
        和prevent一起使用会有问题，第二次点击时，a链接将会跳转
    

v-model 指令: 唯一的数据双向数据绑定指定
    概念引入:
        刚才使用 v-bind {{ msg }}  v-text v-html 这些的都是属于 “单向绑定”，
        我们通过model层（也就是vue对象的data部分），改变其值可以动态地
        绑定到VIew视图上面，但是： 视图上的数据改变并不会影响Model中的数据
        这时就需要双向绑定了
    使用:
        v-model= "变量名"
        使用v-model可以实现表单元素和model中数据的双向绑定
        注意: 只能运用在表单元素中
<div id="app">
    <h4> {{ msg }}</h4>
    <!--v-bind只能实现数据的单向绑定，从M自动绑定到V中去,无法实现数据的双向绑定-->
    <!--<input type="text" v-bind:value="msg" style="width:100%">-->
    <input type="text" style="width:100%" v-model="msg">
</div>

Vue中使用样式:
    1.使用class样式
        1.数组数据
    <h1 :class="['thin', 'red']">这是一个很大很大的h1,大到你无法想象</h1>
        使用v-bind来绑定，由于属性中的内容都将被当做js来执行，所以这里
        将被当做一个 数组 来看待 ['thin', 'red']
        2.有了上面的基础，我们可以写出更加灵活的形式
            比如使用三元表达式:
<h1 :class="['thin', 'red', flag ? 'active': '']">这是一个很大很大的h1,大到你无法想象</h1>
<script>
    var vm = new Vue({
        el: '#app',
        data: {
            flag: false
        },
        methods: {
        }
    })
</script>
        3.在数组中使用对象来代替三元表达式,比上述三元表达式更加简便的一种形式
<h1 :class="['thin', 'red', {'active': flag}]">这是一个很大很大的h1,大到你无法想象</h1>
            当对象中active的值为true，则应用这个类class名，否则不应用
            从这里也可以看出，并不完全是将属性值的内容当做js来执行。。
        4.直接使用对象
            在为class属性使用v-bind绑定对象的时候，对象的属性是类名，对象
            的属性可带引号，也可不带,属性的值是一个标识符
        
    通过style属性写内联样式:
        1.通过:style 指定样式,样式名称无序引号，如果样式名称含有-，则需要引
            号
        
<h1 :style="{color: 'red', 'font-weight': 200}">这是一个h1</h1>

        2.将样式设置给vue变量，然后通过vue变量设置内联样式
<h1 :style="{color: 'red', 'font-weight': 200}">这是一个h1</h1>
        3.设置多套样式给vue变量，然后通过数组来给元素设置多套样式
<h1 :style="[styleObj2, styleObj1]">这是一个h1</h1>

 v-for使用的注意事项:
    循环生成html元素时，应该给元素在绑定一个唯一的key,否则在某些时候,会有问题，比如:
    
<div id="app">
    <div>
        <label for="">Id
            <input type="text" v-model="id">
        </label>
        <label for="">Name:
            <input type="text" v-model="name">
        </label>
        <input type="button" value="添加" @click="add">
    </div>
    //注意这里必须要指定key为一个唯一的值，比如id,用于标识它
    //否则，如果我们选中了前面的复选框，然后再创建一个元素的时候
    //那么复选框的位置将仍然按照原先的索引选中，会不符合预期
    <p v-for="item,index in list" :key="item.id">
        <input type="checkbox" name="" id="">
        {{ item.id }} ----
        {{ item.name }}
    </p>
</div>
<script>
    var vm = new Vue({
        el: '#app',
        data: {
            id: '',
            name: '',
            list: [
                {id: 1, name: '李斯'},
                {id: 2, name: '嬴政'},
                {id: 3, name: '赵高'},
                {id: 4, name: '韩非'},
                {id: 5, name: '荀子'}
            ]
        },
        methods: {
//            添加方法
            add(){
                this.list.unshift({id: this.id, name: this.name})
            }
        }
    })


v-if:
    根据条件决定元素是否显示(直接删除元素):
<h3 v-if="flag">这是v-if控制的元素</h3>
    有较高的性能切换消耗(不停的重新的创建元素)
v-show:
    根据条件决定元素是否显示（改变display为none或者去除none）
<h3 v-show="flag">这是v-show控制的元素</h3>
    有较高的初始性能消耗(初期给其添加style属性)

如果一个元素经常变动，则使用v-if
如果一个元素可能就一次被隐藏，然后之后都不会再出现，那么就使用v-show
