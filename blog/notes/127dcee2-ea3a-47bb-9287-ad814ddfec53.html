1.闭包习题:
    
        function test(){
            arr = [];
            for(var i=0; i<10; i++){
                arr[i] = function(){console.log(i)}
            }            
            return arr;
        }

        myArr = test();
        for(i=0; i< myArr.length; i++){
            myArr[i]();
        }

        结果会输出什么？

        答案是: 10 10 10 10 10 10 10 10 10 10(10个10)
        由于数组中每个函数将内部的作用域保存了出来，事实上10个函数共用了一个test的作用域，并且共用了其中了i变量，所以10个函数中输出的都是同一个值，此外由于循环结束时，i最后还会++一次，所以i等于10.

        如果我们希望这个test返回的数组按照我们预期的 0 1 ..9来输出呢？
        两种方法：
            采用立即执行函数:
            function test(){
                arr=[];
                for(i=0; i<10; i++){
                    (function(j){
                        arr[j] = function(){console.log(j)}
                        return arr;
                    }(i))
                }
                return arr;
            }


            myArr = test();
            for(i=0; i< myArr.length; i++){
                myArr[i]();
            }
            立即执行函数同普通的函数一样，运行的时候也会创建一个单独的作用域，并且它会真正的立即执行，这个时候会将循环的i值真正的赋值给j,也就是说每次循环里面的j的值是不同的，进而实现定义arr[j]函数的时候，函数体中的值也是不同的了。
            知识点在于： 首先，在test的循环中定义的函数，并不会在test函数定义完之后就立马定义，而是需要等到test函数执行的时候，其里面的内部函数才会执行函数定义，再其次，内部函数在定义的时候，其使用的i是一个公用的变量i,所以导致数组中定义的10个函数中的i，实际上都是来自于同一个test函数的AO对象中的公用变量/属性，所以最终输出的值自然也是一致的。而如果使用立即执行函数的话，当执行循环体的时候，每次都会新建一个新的执行期上下文，将作为实参赋值为其中的形参j，这个就相当于在内部的立即执行函数的AO中增加了一个属性，该属性的值为当时具体的i值，所以能够正确到我们预期的i，当然也正因为这是个“立即执行函数”所以才有这样的效果,因为它在执行函数内代码的同时，又有函数会创建自己独立AO对象的特性。如果把立即执行函数换成普通的函数定义就不行了，因为那样定义的函数，它内部的代码根本就不会执行。

                for(i=0; i<10; i++){
                    arr[i] = function(){
                        var k = i

                }
                return arr;

2.复习函数以及立即执行函数：
    函数有2种定义方式，函数声明和函数表达式

    立即执行函数：只执行一次的函数，定义完之后立即执行
    (function(){函数体}())
    后面的()是执行运算符，它可以直接执行表达式，,即直接执行函数定义并且调用函数计算它的值，执行完之后就销毁函数
    立即函数也可以这样定义  var a = function(){}();但是函数执行完之后a将不会继续存储函数的引用，因为函数已经销毁
