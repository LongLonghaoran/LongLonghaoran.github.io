3.computed和 watch的区别?
        其实二者完全是不同的功能，不应去比较区别，不过watch有的时候也可以实现相同的功能，但是底层都是基于vm.$watch实现
        $watch方法:
        
vm.$watch(
            'name',
            (n, o)=>{
                console.log(n, o)
            },
            {
//                lazy为true就是computed的实现
                lazy: true,
                如果监控的值是一个对象的话，那么对象中的key-value的value发生变化也会监控
                deep: true
            }
        )

    computed相比watch:
        computed写法更简洁
    
    computed相比methods:
    computed会有一个缓存的效果，只有当computed依赖的属性发生变化时才会重新执行计算函数
    而methods方法是没有缓存的,每次变更值都会执行函数

    computed内部实现一个”脏值检测“，给computed属性设置了一个dirty属性，每当输出一次computed属性之后，将dirty设为false,之后每次取值判断dirty,如果为false就不执行计算了(说明依赖值未被改动),当依赖的值发生变更后，会将dirty设为true,那么下次再取computed的时候，就会重新执行计算了
