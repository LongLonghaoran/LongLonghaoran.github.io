1.使用watch监听文本框数据的变化
    watch也是vue实例化时的属性之一,使用它可以监视data中数据的变化，数据一旦变化就会触发watch中和属性
    同名的处理函数
    1.监听data属性
    <div id="app">
        <input type="text" v-model="firstname"> +
        <input type="text" v-model="lastname" > =
        <input type="text" v-model="fullname">
    </div>
    <script>
        var app = new Vue({
            el: "#app",
            data: {
                firstname: "",
                lastname: "",
                fullname: ""
            },
            methods: {
            },
            watch: {
                // 使用该属性可以监视data中指定数据的变化，然后触发这个watch中的事件
                // 监听的data上的firstname属性的改变
                // 有2个参数，一个是改变后的值（新值），一个是改变前的值（旧值）
                firstname: function(newVal, oldVal){
                    this.fullname = this.firstname + " " + this.lastname
                },
                lastname(newVal, oldVal){
                    this.fullname = this.firstname + " " + this.lastname
                }
            }
        })
    </script>
    2.监听路由地址的改变
直接监听 '$route.path'即可
    
2.使用计算属性computed
    使用computed属性，定义一些属性方法，它类似于方法，但是在使用它的时候，可以直接将它当做
    属性来使用，“使用的时候不需要加()小括号去调用”,只要计算属性内部中任何使用到的数据发生了变化，都
    将会触发计算属性的重新计算
    特点:
        computed属性会对计算出来的值进行缓存,也就是说我们在视图中多次调用同一个计算属性时，计算属性方法
        也只会执行一次

3.watch computed methods三种方式的对比:
    1.相同点，三者都是function 方法
    2.不同:
        1.computed结果会被缓存，除非依赖的属性变化才会重新计算，主要当做普通属性使用
        2.methods方法表示要给具体的操作，主要书写业务逻辑    
        3.watch可以监听特定数据的变化，从而进行特定业务逻辑操作，可以看做是computed methods的结合体

4.nrm的安装使用
    一个用于解决安装npm包速度太慢的问题的快速切换安装源的包
    说白了，它就是维护了一份安装地址而已，可以让我们从不同源去写在包
    
用法：
nrm ls 列出所有可用的源
nrm use 源的名称
接下来就可以使用该源来安装包了
npm i 包


5.Webpack !!! 开始
    网页中常见的静态资源:
  1. .js .jsx(react中的) .coffee .ts
  2. .css .less .sass .scss
  3. .jpg .png .gif
  4. .svg .ttf .eot .woff
  5. .ejs .art-template .vue(这也是webpack中定义组件的方式)
    由此导致的问题: 
        1.当资源多了，二次请求也就多了，加载速度也就慢了
        2.而且还有错综复杂的依赖关联(比如bootstrap依赖jquery)
    解决方案:
        1.合并，压缩，精灵图、图片采用base64
        2.采用requireJs
    那么，如何完美？地的解决上述问题:
        1.使用gulp
            基于任务的，适合于小项目
        2.使用webpack
            基于项目进行构建，整个项目都会构建
    
    webpack是前端的一个项目构建工具，它是基于Node.js开发出来的一个前端工具
    
    webpack最基本的使用:
        1.全局安装
                npm i -g webpack
                截止2019-11-11,webpack最新版本4.4.11,视频中webpack3才刚刚发布
        2.最基本使用，安装的3.6.0
            windows上执行
            webpack .\src\main.js -o .\dist\bundle.js 将main.js编译为bundle.js
            其中我们在bundle.js中引用了 通过 npm安装的 jquery包
            main.js中的代码如下:
// 这个main.js是我们项目的js入口文件
//1.导入jquery
// 从node_modules中导入jquer包，使用$来接收,相当于requirejs中的 const $ = require('jquery');
// 这是es6中导入模块的方式
import $ from 'jquery';
$(function(){
    $("li:odd").css("background", "lightblue");
    $("li:even").css("background", function(){
        return `#d97634`;
    });
});
        编译完成之后，webpack会生成bundle.js，这个文件中就将所有的js代码全部编译为一个js文件了，我们
        在页面中就直接引入这个编译完成的bundle.js文件来使用
        
        需要注意的，上面的import from导入模块的方式也可以替换为 const $ = require('jquery')，webpack会帮
        我们做好所有的编译工作
