2.清除浮动
    1.浮动带来的影响：
        在标准流中，如果有个嵌套的父子级的盒子，父级盒子如果不指定高度，会被子级盒子的高度撑高。但是，如果子级盒子是浮动的，那么父级盒子将不受影响,高度为0，这个时候与父级盒子同级的兄弟盒子会跑到父级盒子的位置，造成混乱。

        所以，清除浮动的目的就是为了解决父级元素因为子级元素浮动而造成的父级元素高度为0的问题。(在现实情况中很难去明确指定父级元素的高度的),清除浮动更准确的说是清除因为浮动带来的影响

    2.清除浮动的方法
        为什么要清除浮动?
            如果子元素有浮动的话，那么父元素的高度是不会被撑高的
        1.额外标签法
            在最后一个浮动标签后面添加一个标签，并添加清除浮动的属性:
            clear: left\right\both 禁止元素左/右/两边有浮动元素
            优点，通俗易懂，书写方便
            缺点：结构化很差，每次有浮动标签都得加一个
        2.overflow
            给父元素添加overflow: hidden/scroll
            .father {
                overflow: hidden
            }
        3.伪元素清除浮动
            给父级元素加如下几个css属性
            .clearfix:after { /*正常浏览器 清除浮动*/
                content: "";
                display: block;
                height: 0;
                clear: both;
                visibility: hidden;
            }

            ie6清除浮动的方式:
                .clearfix {
                    *zoom: 1
                }
                tip: 属性前面加*，则只有ie6会执行这个属性，其余浏览器会无视。

        4.双伪元素清除法:
            .clearfix:after, .clearfix:before {
                content: "";
                display: table;
            }

            .clearfix:after {
                clear: both;
            }
        
            .clearfix {
                *zoom: 1;
            }
3.定位
    元素的定位属性主要包括定位模式和边偏移两部分
    定位模式:
        position: static 自动定位
                  relative 相对定位，相对于其原文档位置定位
                  absolute 绝对定位，相对于一个已经定位的父元素
                  fixed 固定定位,相对于浏览器窗口
    边偏移:
        偏移值,有4个属性可以配置:
            top right bottom left


3.相对定位(css三大难点:盒模型 浮动 定位)
    * 设置position: relative,然后设置 top right bottom left值
    * 相对定位就是盒子相对于自己定位的左上角进行位移
    * 虽然偏离的位置，但是其占用的位置不会改变

4.绝对定位
    和相对定位类似，也是进行偏移
    * 若所有父元素都没有定位，则相对于当前浏览器窗口进行定位,如果父亲有定位(即便没有指定偏移值)，则相对于最近父级元素进行定位。
    * 元素变为绝对定位后，就不占位置了

5.子绝父相
    * 子级是绝对定位的话，父亲一般是相对定位：如果希望给子级元素在父级元素内进行绝对定位的话，父级必须给定位属性，否则子级绝对定位会对浏览器窗口进行定位，但是父级如果是
    绝对定位的话，那么父级又不占用空间，那么父级的兄弟元素会顶替它的位置。

    * 浮动并非绝对的“脱标”，绝对定位才是真正的脱标,对于div而言，如果div内有文字的话，那么就算div给了浮动，其后面的兄弟元素还是无法来顶替它的位置。

6.定位的盒子居中对齐:
    加了定位/浮动的盒子,margin: 0 auto,就失效了,所以引出一个问题，定位的盒子如何居中？直接写top left数值是不实际的

    1.首先 可以设置 left: 50%，可以偏移到父元素的一半宽度
    2.给出margin-left: 盒子宽度的一半

7.固定定位fixed
    类似于绝对定位，只不过它是相对于整个浏览器窗口定位，
    与绝对定位不同，即便父亲有相对定位，它也仍然按照浏览器
    窗口进行对齐

    * 元素模式转换，使用固定定位后，会隐式将其转换为行内块，
    行内块如果不指定高度默认是根据内容的宽度决定的;类似还有float: left,举个例子,div style="position: fixed",这样子
    div就看不见了，原因是盒子本身没有指定宽度而且里面
    没有内容。,注意： 背景图片不会撑开盒子，插入图片(img)会撑开盒子

    * position: fixed, position: absolute, float: left

8.z-index叠放顺序:
    考虑这样一种，情况3个div,w100 h100,
    position全部为absolute,这样子，三个盒子会叠到
    一起， 最后一个盒子会显示在最上面。
    通过改变 z-index: x，即可改变层叠顺序，默认情况下
    z-index值为0.

    * 只有定位了元素可以使用z-index

9.例子： 去除盒子间的“2倍”边框
    存在这样子一种情况，
        style:
            div {
                w100
                h100
                bgc pink
                float left
                border 1px solid black
            }
        html:    
            div.d1
            div.d2
    这样子两个盒子之间的边框会有2px，要怎么使边框为1px呢?
    答案是:
        div {
            margin-left: -1px;
        }
