  • [图片/附件]

    先git pull,再使用run.sh脚本将配置文件
  • 3.等待正式环境更新
  • 4.在CSC、CED大群发消息: @所有人 CSC优化版本预计19：00发布，请大家提前做好工作安排，大约需要40min 发布完成后通知: 发布完成，请继续使用

在测试环境调用AI回复

  • 在base_create.rb中大约499行的位置,增加一个测试环境的判断
def should_ai_reply?
      return false unless Rails.env.production? || Rails.env.sit?
      ...
end
  • 在ai_reply_service.rb中大约102行的位置,将调用AI的ticket_id的格式调整一下,因为测试环境生成的ticket_id肯定比生产环境小的多而且id肯定是生产环境使用过的，而AI会根据ticket_id判断是否进行过AI回复，如果是回复过的ticket_id则AI会拒绝回复
ticket_id: "test-#{ticket.id.to_s}"

日常运维

  • 1.删除unsent 邮件,如果有邮件发送失败，在sidekiq中看到unsent邮件的重试任务
tr =  TicketResponse.find 32336171
t = tr.ticket
u =t.user
# u = User.find_by email: 
#  t.user = u; t.save
s = Mailman::Sender.new tr
s.send :make_mail
#s.send :send_via_google_smtp
#s.send :send_via_mailgun
#s.send :send_via_rakuten
s.send :send_via_ses
  • 2.对于用户自己填写错误邮件地址导致发送失败的重发邮件
# 在sidekiq retry中找到 400 Bad Request: 'to' parameter is not a valid address. please check documentation类似这样的报错,取出参数 ticket_response_id
tr =  TicketResponse.find 14603945
t = tr.ticket
u =t.user
u.update(email: 正确的邮箱地址)
# 在sidekiq面板上点击retry
  • 如果user的email出现重复，无法更新(系统已有同名邮箱用户),则需要将此用户删除,并将与之关联的ticket customer_profile都绑定到正确的user上去
u2 = User.find_by_email 正确的邮箱地址
u.tickets.update_all(u: u2, customer_profile_email: u2.email)
u.customer_profile.update(email: u2.email)
  • 如果邮箱地址明显是乱填的
    则将ticket_response的delivery_status改为 :skip_send
tr.update delivery_status: :skip_send

新增页面时配置权限

RBAC:

  1. 进入http://uat-rbac.anker-in.com.cn/#/permission，切换到CSC系统，参照其他权限新增一个权限配置
  2. 进入roles界面，为需要添加该权限的角色添加权限，比如admin一般是所有的权限，可以从admin开始添加
