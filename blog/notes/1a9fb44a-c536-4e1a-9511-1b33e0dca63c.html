RDMA网卡面向NS有两种模式：

NS共享模式（默认，可以通过修改驱动参数使默认模式为独占模式）

NS独占模式

查看当前模式：

rdma system

修改RDMA网卡模式（如要修改模式，需要在创建其他NS之前就进行配置）：

rdma system set netns exclusive
rdma system set netns shared

其中RDMA命令是由iproute包提供，当前SP2的iproute版本：5.5.0、内核版本：4.19.90-24.4.v2101.ky10无法使用上面的命令，会异常报错。

后部署MLNX_OFED的驱动包，可以正常使用该模式切换命令。

1.1.2. 同节点跨NS

1.1.2.1. MACVLAN

ip netns add one
ip link add one_mac link <eth0> type macvlan mode bridge
ip link set one_mac netns one
 
ip -n one address add 100.100.250.1/24 brd + dev one_mac
ip -n one link set one_mac up
 
ip netns add two
ip link add two_mac link <eth0> type macvlan mode bridge
ip link set two_mac netns two
 
ip -n two address add 100.100.250.2/24 brd + dev two_mac
ip -n two link set two_mac up

1.1.2.2. VETH

ip netns add one
ip link add one type veth peer name one_veth netns one
ip link set one up
ip -n one link set one_veth up
 
ip -n one address add 100.100.100.1/24 brd + dev one_veth
ip -n one route add 169.254.1.1 dev one_veth
ip -n one route add default via 169.254.1.1
ip -n one neigh add 169.254.1.1 lladdr $(ip link show dev one | tail -n +2 | awk '{print $2}') dev one_veth
ip route add 100.100.100.0/24 dev one
 
ip netns add two
ip link add two type veth peer name two_veth netns two
ip link set two up
ip -n two link set two_veth up
