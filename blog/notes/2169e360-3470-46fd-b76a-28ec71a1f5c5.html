屏幕的触摸事件
    touch是移动端的触摸事件，而且是一组事件
    touchstart 当手指触摸到屏幕时就触发
    touchmove 当手指在屏幕中滑动时触发
    touchend  当手指离开屏幕时触发
    touchcancel 当被迫终止滑动的时候触发(比如由于其他应用的弹窗，比如来电话了？导致页面触摸事件被终止)
    利用touch相关事件就可以实现移动端常见滑动效果和移动端常见的手势事件
    绑定事件，我们只需要使用addEventListener就可以了，因为我们只需要考虑谷歌浏览器的情况就够了
    js会将触摸的事件对象作为参数传递事件处理函数
        addEventListener(dom, function(event){})
        其中event就是触摸事件对象，我们需要关注其中的3个属性:
        1.changeTouches    类型 TouchList,可以称它为触摸点的集合(一个手指触摸就是一个触发点，2个手指就是2个触发点）
            改变过的触摸点集合，所谓改变过指的就是，比如 手指初次触摸到屏幕，就会有触摸点从无到有
        2.targetTouches    类型 TouchList,可以称它为触摸点的集合
            当前元素的触摸点集合
        3.touches          类型 TouchList,可以称它为触摸点的集合
            页面上所有的触摸点集合，在touchend事件中，由于手指已经离开屏幕，所以targetTouches和touches
            集合都将为空，只有changedTouches不为空
    触摸点:
        需要关注3组属性
        clientX clientY(基于浏览器窗口，而且由于我们已经将视口调整为和页面大小一致，所以也可以理解为基于视口）
        pageX pageY （基于页面，而由于页面和浏览器窗口大小是一致，所以也可以说是基于视口 基于窗口）
        screenX screenY 基于整个屏幕，这个就和前两者不同了，基于屏幕意味浏览器窗口以外的距离也会算进来
    如果手指只是点击了一下屏幕，并不会触发touchmove,但是touchstart touchend都会触发
轮播图的滑动功能
    原理：通过监听手指滑动屏幕的事件，来同时改动图片的偏移值（比如left rigth，但是我们这里采用的是transform属性的translateX)
    1.通过touchstart事件，记录手指刚刚碰触屏幕时的x位置，通过event.touches[0].clientX来获取
    2.通过touchmove事件，在其中继续通过event.touches[0].clientX获取手指实时的位置，并计算手指在屏幕滑动的距离
    3.通过touchend事件，判断从手指开始触碰到手指离开屏幕总共移动了多少距离，然后决定轮播是否移动，以及朝哪个方向移动

关于移动端的滑动手势:
    通过touch touchmove touchend事件，其实已经揭示出，我们可以通过移动的轨迹，来判断用户滑动的手势
移动web中的tap事件(tap 轻点/轻按/轻触)
    所谓轻点/轻触，就是说用户在屏幕上迅速的点击一下，而不是按住不动或者按住之后进行滑动
    其实移动端也有 click事件 ，但是为了区分是滑动还是点击，click事件有个“点击延迟”，如果
    手指在触发到屏幕后，如果在300ms内没有滑动，那么就认为这是个点击事件，会触发click
    但是就是这300ms的延迟，其实也是会影响用户体验的，会有明显的感觉，点击事件延迟了0.3秒才触发
    所以就需要更快的触发点击的事件，我们可以改用tap事件
    tap事件，不是移动端原生事件，通过touch相关事件衍生过来的
        实现代码如下:
            var bindTapEvent = function(dom, callback){
                var startTime;
                var isMove = false;
                dom.addEventListener('touchstart', function(){
                    startTime = Date.now();
                });
                dom.addEventListener('touchmove', function(){
                    isMove = true;
                });
                dom.addEventListener('touchend', function(e){
//                    当时间间隔小于150且未滑动
                    if((Date.now() - startTime)<150 && !isMove){
                        callback && callback.call(this, e);
                    }
                    isMove = false;
                });
                dom.addEventListener('click', function(){
                    console.log('click');
                });
            };
    借助一个插件: fastclick.js 也可以提高click响应速度

做法1:
    左边栏绝对定位，并给予固定宽度，右边栏宽度100%,并配置padding-left，其值等于左栏宽度
做法2:
    左边给个浮动定位，右侧常规块级定位，并且给个overflow:hidden,给了overflow:hidden之后,之前也讲过清除浮动的方法
    就是给其父元素添加overflow:hidden,overflow:hidden有一个作用就是使其成为一个绝缘体，会避免其他浮动元素对自身
    造成影响，同时避免因为自身子级元素浮动对其他元素造成影响
