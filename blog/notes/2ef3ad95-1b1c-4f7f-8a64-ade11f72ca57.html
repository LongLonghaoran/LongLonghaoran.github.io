1.document.querySelector document.getElementById的区别:
    querySelectorAll 返回的是一个 Static Node List
    getElementsBy 系列的返回的是一个 Live Node List
    1.querySelector返回的只是一个元素节点的快照，它是静态的
    2.getElementsByTagName返回的是一动态的元素对象集合
    3.举个例子:
        页面上:
            <ul>
                <li></li>
                <li></li>
                <li></li>
            </ul>
        js:
            var objs1 = document.getElementsByTagName("li")
            var objs2 = document.querySelector("li")
            这个时候从页面上删除一个li，再来查看objs1 objs2，会发现
            getElementBy的结果已经变为2了，而querySelector出来的长度
            仍然为3
    4.再来个例子:
        //Demo1
        var temp = document.querySelectorAll(".class");
        for (var i=0, max=temp.length; i<max; i++) {
            temp[i].className = "new_class";
        }
        //Demo2
        var temp = document.getElementsByClassName("class");
        for (var i=0, max=temp.length; i<max; i++) {
            temp[i].className = "new_class";
        }
        假如获取到的temp的长度都为3，
        Demo1能将三个元素的className全部更改为new_class”，
        Demo2只能讲第一个元素和第三个元素的className更改为”new_class”
    5.再来个例子:
        其实除了getElement以外，还有children childNodes这些方法，它们都是动态获取元素的
        这是一个按钮的点击事件，它的作用是将id=dv的div中的所有子元素用循环的方式全部删除，
        但是下面的方法最多只能删除一半的元素(当总数为奇数时，删除一半+1个元素)
        my$("btn3").onclick = function(){
            var dv = my$("dv");
            //var children = document.querySelectorAll("#dv button");
            var children = my$("dv").children;
            var length = children.length;
            for(var i=0;i<children.length;i++){
                console.log("i:"+ i + ",----length:" + children.length);
                if(dv.firstElementChild){
                    dv.removeChild(dv.firstElementChild);
                }
            }
        };
        上面的my$("dv").children获取的永远是实时的子元素集合
        这是由于随着子元素的删除，获取到的子元素的数量也在同时减少
        如果有2n个子元素的话，当删除了n个子元素的时候，子元素的总数也变为了n个,这个时候计数器i++,
        i++==n++且i++ > n，所以就出现这种情况了
        但是，如果使用的document.querySelector("dv")，就不会有这种情况出现了
2.节点:
    文档:document
    元素：页面种所有标签  Element
    节点：页面种所有的内容  Node
    根元素：html
    元素是节点，但节点不一定是元素，从大小概念来说，节点>元素

    引入一个问题:
        在一个错乱的页面中，比如都没有id也都没有class，如果还想通过元素选择getElementByxx来获取元素或者节点
        的话，是非常困难的
    那么我可以通过节点的几个属性来方便操作
        nodeType: 节点的类型 1--标签 2--属性 3--文本
        nodeName: 节点的名字，标签节点--大写的标签名字，属性节点--小写的属性名， 文本节点--#text
        nodeValue: 节点的值，标签节点--null,属性节点--属性值，文本节点--文本内容
    1.找到父元素、父节点
        parentElement parentElement
    2.找到子节点、子元素
        childNodes
        计算子节点，只会计算 文本节点和元素节点
        children
        计算子节点只会计算元素节点
    3.获取属性节点(不是很常用):
        1.首先获取一个元素节点
        2.obj.getAttributeNode("属性名") 即可
    4.第一个子节点 第一个子元素
        1.firstChild firstElementChild
    5.最后一个子节点 最后一个子元素
        lastChild lastElementChild
    6.某个元素的前一个兄弟节点 某个元素的前一个兄弟元素
        previousSibling previousElementSibling
    7.某个元素的后一个兄弟节点 后一个兄弟元素
        nextSibling nextElementSibling
    注意点:
        在IE8中但凡是获取元素的方法都有可能不生效，返回一个undefined，
        而且IE8中获取节点的方法获取到的会是元素

5.为元素绑定多个事件
    当我们使用onclick属性来绑定事件时，如果多次使用了onclick属性，那么将只会
    有一个事件被触发，最后绑定的事件将被触发
    如果要绑定多个事件:
        addEventListener(类型, 回调函数, false)第三个参数false可省略
    注意：这个方法在IE8中不支持,IE8中支持attachEvent
    但是attachEvent在firefox和chrome中又不支持,没必要使用attachEvent,因为在
    IE11中也不支持attachEvent

    处理多个浏览器的兼容问题:
        if(document.querySelector("#btn").addEventListener){
            document.querySelector("#btn").addEventListener('click', function(){
                console.log("单击了")
            })
        }else if(document.querySelector("#btn").attachEvent){
            document.querySelector("#btn").attachEvent('onclick', function(){
                console.log("单击了")
            })
        }else{
            document.querySelector("#btn").onclick = function(){
                console.log("单击了")
            }
        }

6.虽然在不支持某个方法的浏览器调用方法会报错，但是如果只点出方法名，是不会
    报错的，而是返回一个undefined，使用这种方法可以用来在浏览器中测试
    是否支持某个方法
    比如:
        attachEvent这个方法，在chrome firefox ie11中都不支持，直接调用
        document.getElementById("btn").attachEvent('onclick', function(){})
        的话会报错: VM20:1 Uncaught TypeError: document.getElementById(...).attachEvent is not a function
                   at <anonymous>:1:32
        但是我们可以通过点方法名来测试，因为只点方法名的就相当于是去点属性，
        在js中，对于不存在的属性，返回值都是undefined
