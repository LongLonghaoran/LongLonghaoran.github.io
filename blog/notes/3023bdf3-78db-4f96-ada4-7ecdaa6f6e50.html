0. 购物车小球案例关键点，通过js钩子函数实现的时候，created函数中直接将flag置为false，那么后半场钩子就不会执行了
Vue把一个完整动画，使用钩子函数，拆分了两部分，在Vue中，钩子函数的默认行为就是:
    1.前半场，flag从false变为true
    2.后半场,flag从true变为false
    
<div id="app">
    <input type="button" value="加入购物车" @click="flag=!flag">
    <transition @before-enter="beforeEnter" @enter="Enter" @after-enter="afterEnter">
        <div class="ball" v-show="flag"></div>
    </transition>
</div>
<script>
    var vm = new Vue({
        el: '#app',
        data: {
            flag: false
        },
        methods: {
            beforeEnter(el){
                el.style.transform = 'translate(0, 0)'
            },
            Enter(el, done){
                el.offsetWidth;
                el.style.transform = 'translate(50px, 250px)';
                el.style.transition = 'all 1s ease'
                done();
            },
            afterEnter(el){
                this.flag = !this.flag;
            }
        }
    })
</script>

1.父子组件传值
    1.Vue实例本身也可以看做是一个组件，在它这个实例内部定义的组件，就相当于是它的子组件
    2.在子组件的模板中是无法直接使用父组件中的data数据的
    3.父组件引用子组件时，通过v-bind的方式，将数据传递到子组件内部
    4.子组件如果想要使用传递过来的属性，必须在组件的props中定义一个和传递过来的属性名称一样的名称，然后
        就可以利用传递过来的数据
<div id="app">
    <com1 :parentmsg="msg"></com1>
</div>
<script>
    var vm = new Vue({
        el: '#app',
        data: {
            msg: '父组件中的数据'
        },
        methods: {
        },
        components: {
            com1: {
                data: function(){
                    return {
                        msg: '这是子组件自身的数据'
                    }
                },
                template: '<h1>这是子组件--{{parentmsg}}--{{msg}}</h1>',
                props: [
                        'parentmsg'
                ]
            }
        }
    })
</script>
    5.通过父组件传递过来的数据是只读的,不可修改

2.子组件通过事件调用向父组件传值
    1.父组件向子组件传递方法，采用的是事件绑定机制; v-on/@
        @自定义事件属性名称='父组件中的方法名'
    2.在子组件中，通过 this.$emit('自定义的事件属性名称')， 来调用父组件传递过来的方法
<template id="tmpl">
        <div>
            <h1 >这是子组件</h1>
            <button @click="myClick">这是子组件中的按钮</button>
        </div>
    </template>
    <div id="app">
        <com2 @func="show"></com2>
    </div>
    <script>
        var com2 = {
            template: '#tmpl',
            methods: {
                myClick(){
//                    emit英文原意：触发、调用、发射
//                    通过emit可以触发父组件传递过来的方法，将方法名以字符串作为参数传递
                    this.$emit('func')
                }
            }
        };
        var vm = new Vue({
            el: '#app',
            data: {
                msg: '父组件中的数据'
            },
            methods: {
                show(){
                    console.log('调用了父组件身上的show方法')
                }
            },
            components: {
                com2
            }
        })
    </script>
$emit的第二个开始的参数就作为父组件中原方法的参数
    
3.$ref 获取DOM元素和组件引用
    1.获取dom元素
        1.在需要引用的dom元素上添加 ref='名称' 属性
        2.在vue实例内部就可以通过 this.$refs获取到元素，键就刚才我们自定义的名称
    2.获取组件

4.Vue中的路由
    概念引入:
        对于单页应用程序来说,主要通过url中的hash(#号)来实现不同页面之间的切换,
        同时,hash有一个特点:http请求中不会包含hash相关的内容,所以，单页面程序中的页面跳转
        主要用hash实现
    在单页程序中，通过hash来切换页面的方式，称作前端路由


5.Vue中实现路由，使用的vue-router,这是官方推荐的路由模块
    使用方式:
        1.新建路由对象 new Router
        2.给构造函数传入参数，传递一个对象作为参数，对象中通过routes指定路由   
        3.routes是一个数组，每个元素表示一个路由规则
        4.每个路由有2个必要的属性 path 匹配的路径, component 路径匹配后展示的组件
        5.在vue实例的选项中通过router来注册这个路由对象
    var login = {
        template:'<h1>登录组件</h1>'
    };
    var register = {
        template:'<h1>注册组件</h1>'
    };
//    在New路由对象的时候，可以为构造函数传递一个配置对象options
    var routerObj = new VueRouter({
        //这个配置对象中的route表示路由匹配规则
        routes: [
//                每个路由规则，都是一个对象，这个对象身上，有2个必须属性： path 表示监听哪个路由地址, component 表示，如果路由是前面匹配的path,则展示component属性对应的组件
            {
                path: '/login',
//                这里直接将模板对象放进去,不需要用引号包裹组件名称
                component: login
            },
            {
                path: '/register',
                component: register
            }
        ]
    });


//    创建一个路由对象
    var vm = new Vue({
        el: '#app',
        data: {},
        methods: {},
//        通过router属性来将路由注册进来
        router: routerObj
    });
    6.在vue容器放置一个路由容器 router-view:    
<div id="app">
//这是vue-router提供的元素
    <router-view></router-view>
</div>
    7.这个可以给页面添加两个a链接，用于切换组件
//注意url以#开始，这样地址就会被当做锚点来处理，不会执行页面跳转，而是在当前页面的url尾部将锚点(#部分)替换
<a href="#/login">登录</a>
<a href="#/register">注册</a>

6.使用router-link创建链接
    除了使用a标签创建链接，还可以通过router-link创建链接,通过to属性指定要跳转的组件，通过to执行组件可以 
    不用在前面写 #号，直接写路由路径即可
    router-link还有tag属性，可以指定生成元素所用的标签，也就是说它可以使用其他元素通过也实现 a标签的功能

7.使用redirect重定向
    路径规则第二个参数使用redirect: '组件名称',可以将指定要重定向到的组件，注意：此重定向并非浏览器的重定
    向，只是类似重定向而已

8.路由的类名    
    1.通过router-link生成的链接，会自动的被加上 router-link-exact-active router-link-active 这2个类
        我们直接给这两个类加样式就可以高亮显示当前被选中的链接了
    2.我们可以自定义这个类名，通过路由构造函数的配置项中的linkActiveClass来配置即可

9.路由传参        
    就像是服务端通过url或者request body接收参数一样，vue-router中的路由也能够接收参数
    1.使用query方式
        1.直接在url后面加上查询字符串 比如 ?id=10
        2.在组件内部，直接通过 this.$route.query即可拿到参数
        3.其中 this.$route 是挂载在这个组件上的路由对象
    2.使用params方式
        1.在路由规则中，通过:参数名称 的形式，来传递参数，比如 /login/:id
        2.通过params获取参数
            this.$route.params
            这个params就是参数名称-值 组成的一个对象，这个参数对象是通过 router中的matched来获取的，
            vue-router内部使用就是正则表达式去匹配路径以此来获取参数的
        3.以这种方式可以写任意多个 : 占位符，用于传递多个参数

10.路由的嵌套
    场景引入:    我们写了一个 用户组件，用于展现 用户的登录和注册界面
                        其中登录和注册的a链接 又写在 作为父组件的 用户组件中，我们希望单击这两个超链接时，
                        路径能够跟在父组件的路径之后,比如 /account/login /account/register
        1.通过chlidren属性实现，在children中继续书写路由规则，注意：子路由的path不要以 / 斜杠开头，
            这样子路由匹配的路径是以 父路由为前缀的，比较好理解
<div id="app">
    <router-link to="/Account">Account</router-link>
    <router-view></router-view>
</div>
<template id="tmpl">
    <div>
        <h1>这是Account组件</h1>
        <router-link to="/Account/login">登录</router-link>
        <router-link to="/Account/register">注册</router-link>
        <router-view></router-view>
    </div>
</template>
<script>
    var account = {
        template: '#tmpl'
    };
    var login = {
        template: '<h3>登录</h3>'
    };
    var register = {
        template: '<h3>注册</h3>'
    };
    var router = new VueRouter({
        routes: [
            {
                path: '/Account',
                component: account,
                children: [
                    {path: 'login', component: login},
                    {path: 'register', component: register}
                ]
            }
//                不能这么写
//            {path: '/Account/login', component: login},
//            {path: '/Account/register', component: register}
        ]
    });


//    创建一个路由对象
    var vm = new Vue({
        el: '#app',
        data: {},
        methods: {},
//        通过router属性来将路由注册进来
        router: router
    });
</script>
