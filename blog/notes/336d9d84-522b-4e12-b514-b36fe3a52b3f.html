2.关于php的api
    php的功能几乎全是依靠它大量的api(而不像ruby本身基础语法就很强),php有1000多个内置函数
    php中的api结构:
        宽字符处理函数:
            strlen是获取拉丁文长度的一个函数，而中文是属于宽字符
            php专门为宽字符添加了一套api,以 mb_ 开头的,比如 mb_strlen,但是它不在默认的1000多个api中，
            在一个扩展库中,叫做 mb_string.dll,模块成员必须通过配置载入模块过后再使用
            其实在php安装后它已经被包含在安装目录的ext目录，但是并没有启用，还需要我们增加扩展:
            php中配置扩展:
                1.安装目录中的php.ini-development, 将它改名为php.ini,然后在编辑中打开编辑它
                    调整extension_dir = xxx这个行，更改至扩展目录
                2.解除extension=mbstring的注释(注释是;)
                3.修改php默认的php.ini配置的路径(默认是 C:\Windows),将我们刚才修改好的php.ini文件
                    移入到C:\Windows下即可
                3.1 但是并不建议这样做，应该是修改Apache的配置文件来调整路径
                    在 httpd.conf 中使用配置选项
                    PHPIniDir php.ini所在路径
            php的repl (“读取-求值-输出”循环（英语：Read-Eval-Print Loop，简称REPL）是一个简单的，交互式的编程环境)
                在php安装目录下有个 php.exe,可以在命令行中使用它像ruby irb一样编写php代码
                输入 php -a 启动交互模式,注意 php中代码一定要以;分号结束才行
            isset 一个变量是否有定义或者一个值是否为Null?
                数组如果引用了不存在的索引，则返回为null,那么isset函数就返回false,否则返回true,
                而且有一点可以注意下，如果数组引用了不存在的索引是会报一个 Notice: xxx的，而如果
                直接将$arr['不存在的索引']作为参数给isset函数的话，那么Notice会被隐藏
                官方:
                    检测变量是否设置，并且不是 NULL。
                    如果已经使用 unset() 释放了一个变量之后，它将不再是 isset()。若使用 isset() 测试一个被设置成 NULL 的变量，将返回 FALSE。同时要注意的是 null 字符（"\0"）并不等同于 PHP 的 NULL 常量。
                    如果一次传入多个参数，那么 isset() 只有在全部参数都以被设置时返回 TRUE 计算过程从左至右，中途遇到没有设置的变量时就会立即停止。
            empty:
                检查一个变量是否为空,为空返回true,就是说它的返回结果与isset相反
                官方:
                    判断一个变量是否被认为是空的。当一个变量并不存在，或者它的值等同于FALSE，那么它会被认为不存在。如果变量不存在的话，empty()并不会产生警告。
                    参数
                        var
                    待检查的变量
                    Note:
                        在 PHP 5.5 之前，empty() 仅支持变量；任何其他东西将会导致一个解析错误。换言之，下列代码不会生效： empty(trim($name))。 作为替代，应该使用trim($name) == false.
                        没有警告会产生，哪怕变量并不存在。 这意味着 empty() 本质上与 !isset($var) || $var == false 等价。
                    返回值
                        当var存在，并且是一个非空非零的值时返回 FALSE 否则返回 TRUE.
                        以下的东西被认为是空的：
                        "" (空字符串)
                        0 (作为整数的0)
                        0.0 (作为浮点数的0)
                        "0" (作为字符串的0)
                        NULL
                        FALSE
                        array() (一个空数组)
                        $var; (一个声明了，但是没有值的变量)
                补一个js知识:
                    两个变量相比较时，会将 == 两边转换为 数字再来比较，比如: '0' == false //true,两者转为数字都是 0
            date和time函数:
                可以参看官方文档，更加详细
    超全局变量:
        它的作用域范围就类似之前的全局变量,因为在php中全局变量还不能被局部作用域访问到，需要使用 global声明一下才可以访问，
        而超全局就是不用声明也可以访问
    常量:
        php中可以使用define定义一个常量，常量的特点是，定义后不能被修改，也是临时存放数据的容器,
        一般用于程序的配置类的信息
        define('大写的蛇底式常量名', '值')
        比如: define('SYS_NAME', '阿里百秀'， 常量是否忽略大小写(最新的php版本已经废弃，现在无法忽略大小写))
        常量可以直接用名称来调用(php中的变量 要加上 $美元符号来使用)
    require 载入文件:
        引入一个文件并且立即执行，类似于css中的 @import url("另一个css文件的路径"）
        特点:
            1.重复载入可能会出问题(比如载入的文件中定义了常量，常量重复定义的话会报错)
            2.还可以引入html代码,php文件中直接html,将共用的Html页面分出去
            3.如果载入的文件不存在，会报错,并且将导致require后面的代码全部失败,不会执行
        require_once:
            特点:
                1.只会载入一次
    include 载入文件:
        特点:
            1.如果载入的文件不存在，虽然也会出错，但是不会影响其后的代码，只会给一个警告错误
        include_once:
            只会载入一次
    综合起来：
        一般像一些重要的配置文件，比如数据库配置文件，就需要用require_once
        像页面的一部分html，就可以用include_once，即使缺一部分也不要影响其他部分
