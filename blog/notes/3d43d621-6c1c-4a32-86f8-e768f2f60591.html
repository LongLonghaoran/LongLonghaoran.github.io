1.作用域 重点
    全局变量:
        除了在函数内部声明以外，剩下的任何任何位置定义的变量都是全局的,在任何位置都能访问
    局部变量:
        在函数内部定义的变量是局部变量
        function() {
            var num=10;
        }
    隐式全局变量：声明的变量没有用var,就是隐式的全局变量

    delete: delete用于变量,但是只能删除隐式的全局变量,因为被var定义的变量会被标记
        为不可配置的，所以无法删除
        MDN:
        // 在全局作用域创建 adminName 属性
        adminName = 'xyz';

        // 在全局作用域创建 empCount 属性
        // 因为我们使用了 var，它会标记为不可配置。同样 let 或 const 也是不可配置的。
        var empCount = 43;

        EmployeeDetails = {
          name: 'xyz',
          age: 5,
          designation: 'Developer'
        };

        // adminName 是全局作用域的一个属性。
        // 因为它不是用 var 创建的，所在可以删除。
        // 因此，它是可配置的。
        delete adminName;       // 返回 true

        // 相反，empCount 是不可配置的，
        // 因为创建它时使用了 var。
        delete empCount;       // 返回 false

        // delete 可用于删除对象的属性
        delete EmployeeDetails.name; // 返回 true

        // 甚至属性不存在，它也会返回 "true"
        delete EmployeeDetails.salary; // 返回 true

        // delete 对内建静态属性不起作用
        delete Math.PI; // 返回 false

        // EmployeeDetails 是全局作用域的一个属性。
        // 因为定义它的时候没有使用 "var"，它被标记为可配置。
        delete EmployeeDetails;   // 返回 true

        function f() {
          var z = 44;

          // delete 对局部变量名不起作用
          delete z;     // 返回 false
        }
    delete和原型链:
        在下面的示例中，我们删除一个对象的自己的属性，而原型链上具有相同名称的属性可用：
            function Foo() {
              this.bar = 10;
            }

            Foo.prototype.bar = 42;

            var foo = new Foo();

            // 返回 true，因为删除的是 foo 对象的自身属性
            delete foo.bar;

            // foo.bar 仍然可用，因为它在原型链上可用。
            con sole.log(foo.bar);

            // 从原型上删除属性
            delete Foo.prototype.bar;

            // 输出 "undefined"，因为不能继承这个属性了
            console.log(foo.bar);

    全局作用域: 即 全局变量的作用域（在函数之外定义的变量或者，函数内没有var的变量）
    局部作用域  即 局部变量的作用域（在函数定义的变量）

    块级作用域(比如用一对大括号括起来的一块区域): js里面没有块级作用域
2.作用域链 重点
    当调用一个变量的时候，会依次从当前作用域开始寻找该变量的定义，如果一直到最上级都
    找不到，则报错
    如:
        num =1;
        function f1(){
            var num =2;
            function f2(){
                var num =3;
                function f3(){
                    var num=4;
                    console.log(num);
                }
                f3();
            }
            f2();
        }
        f1();
        结果是4，会从距离console.log最近的当前作用域开始寻找

3.预解析/预编译 重点:
    举个例子:
        //我可以在变量声明定义之前就使用该变量
        consoloe.log(num);
        var num=10;
        结果是undefined
    说明，在js中执行代码时，会将变量的声明提前，作用域提升，注意，仅仅
    只是声明提前了而已，赋值操作/初始化并没有提前。
    1.函数和变量的声明都会被提前到当前作用域的最上面。
    2.预编译时，变量的提升，只会在当前的作用域中提升
    3.预编译会分段(多对的script标签中函数声明，预编译的时候不会冲突),但是有个
        特殊的地方,如果是在排在前面的script标签中声明的变量，那么后面的script中
        也能访问到，这个其实不算是预编译，这只是顺序执行代码而已
    4.预解析过程:
        1.把变量的声明提升到当前作用域的最前面，只会提升声明 ，不会提升赋值
        2.把函数声明提升到当前作用域的最前面
        3.先提升var，再提升function
    5.函数的声明总是在最后进行提升的,比如声明了一个同名的函数的变量a，当我们
        访问a时，我们访问的总会是a函数:
        function a(){

        }
        var a;
        console.log(a);//结果时function a(){}
    6.连续赋值操作中,从第二个变量开始，默认都成为隐式全局变量
        例如:
            function f1() {
                var a = b = c = 9;
                console.log(a);
                console.log(b);
                console.log(c);
            }
            f1();
            console.log(c); //9
            console.log(b); //9
            console.log(a); //只有这一行会报错
        函数f1中，a会由于函数的执行结束销毁之后，跟随函数f1一同销毁，所以报错
        但是b c 由于没有使用var进行声明，而是直接赋值，所以它们会转换为
        隐式全局变量，即 window.b window.b 所以即便函数调用结束之后，它们也
        仍然会存在，或者也可以这么解释： b c已经是全局变量了，而a由于是在
        函数这个局部范围内通过var声明为局部变量了，所以在函数外部是访问不到的。
4.面向对象 对象 重点
    面向对象的特性： 封装 继承 多态
    1.创建对象
        1.调用系统的构造函数创建对象
            1.
                var obj = new Object();
                obj.name = '龙哥';
                obj.age = 25;
                obj.eat = function(){
                    console.log("吃饭ing....");
                };
                console.log(obj.name);
                obj.eat();
            2.instanceOf 判断实例是属于哪个对象类型的 obj instanceof Object
            3.this 在当前对象中可以使用this指代当前对象,在对象中this指代当前对象,
             在顶级作用域中this则指代window
            4.如何一次性创建多个对象,把创建对象的代码封装在一个函数中,也叫工厂模式创建对象,
                比如:
                    function createObject(name, age){
                        var obj = new Object();
                        obj.name = name;
                        obj.age = age;
                        obj.sayHi = function(){
                            console.log("我叫" + this.name + ",我今年" + age + "岁了")
                        }
                        return obj;
                    }
                    obj1 = createObject('龙', 25);
                    obj2 = createObject('茹媛', 26);
        2.自定义构造函数创建对象
            1.通过第一种方式创建出来的对象有个很明显的缺陷，不能直接分辨出
            创建的是哪种对象,因为都是通过new Object()创建出来的
            2.自定义构造函数创建对象:
                自定义一个函数，首字母大写（之所以大写，只是为了和普通函数进行区别）
                function Person(name, age){
                    this.name = name;
                    this.age = age;
                    this.sayHi = function(){
                        console.log("我叫" + this.name+ ",我今年" +this.age +"岁了" )
                    }
                }
                var p1 = new Person('龙', 25);
                var p2 = new Person('茹', 26);
                p1.sayHi()
                p2.sayHi();
            3.通过构造函数创建对象时系统所作的4件事:
                在内存中申请一块空闲的空间,存储创建的新对象
                把this设置为当前对象: this = {}
                设置对象的属性和方法的值: this.xx = xx
                返回this:  return this
        3.字面量的方式创建对象
            1.直接用{}包起来新建一个对象:
                var obj = {};
                obj.name = 'ss'
                obj.sayHi = function(){};
            2.它和构造函数 new Object()创建的对象有什么区别？
                其实没有任何区别，某种意义上来说 {}这种字面量的方式，只是js的一种
                语法糖而已
            3.字面量创建对象的缺陷:
                无法像构造函数一样，传参数来给属性赋值，也就是说它创建的是一次性
                的对象
    2.设置和获取属性的另一种方法:
        通过[]中括号
            obj['name'] 等价于 obj.name
        注意: 方法属性也可以这样调用
            var obj = {
                name: '龙浩然',
                play: function(){
                    console.log(this.name + "不想玩，"+this.name+"只想学习");
                }
            }
            obj['play']();
5.json格式数据以及遍历:
    1.json也是一个对象，数据都是成对的，一般json格式的数据无论是键还是值都是用双括号括起来的
    2.虽然键值都是字符串，但是访问属性的时候，点方式 和 []方式都可以使用
        var obj = {
            "a": "1",
            "b": "2"
        }
        obj.a
        obj["a"]
        都是可以的
    3.json数据的遍历
        1.不能通过for循环，因为json是无序的，for循环用于可以使用 arr[0] arr[1] ..arr[index]有序的情况
        2.通过for in来循环
            var json = {
                "name": "小明",
                "age": "10",
                "sex": "男"
            }
            for(var key in json){
                console.log(json[key]);
            }
            其中的key是一个变量，这个变量中存储的是该对象中的属性的名字
6.js中的数据类型:
    1.
        原始数据类型 number string boolean undefined null object
        值类型 number string boolean
        复杂类型 object
        引用类型 object
    2.内存中存储数据时有 栈 跟 堆 两块地方
        栈： 有顺序的存储值，先进先出
        堆： 无顺序的存储
        对于值类型，就是按照顺序将值直接存储在栈中
        对于引用类型，会将引用的对象存储在堆中，将存储的对象的地址存在栈中
        值类型之间传递，如果作为函数的参数，传递的是值
        引用类型之间传递，如果作为函数的参数，传递的是地址
