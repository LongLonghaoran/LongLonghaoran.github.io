1. 使用

在调用任何 DDP 其他方法之前，需要使用torch.distributed.init_process_group()进行初始化。该方法会初始化默认分布式进程组和分布式包。此方法会阻塞，直到所有进程都加入，函数定义如下：

def init_process_group(
    backend: Optional[str] = None,
    init_method: Optional[str] = None,
    timeout: Optional[timedelta] = None,
    world_size: int = -1,
    rank: int = -1,
    store: Optional[Store] = None,
    group_name: str = "",
    pg_options: Optional[Any] = None,
    device_id: Optional[torch.device] = None,
) -> None:

其中重要的参数：

backend：后端通讯库名称，例如：'MPI'、'GLOO'、'NCCL'等

world_size：进程组中的进程数量，当提供`store`时该参数必须提供

rank：当前进程的自身的编号，范围为0到world_size-1，当提供`store`时该参数必须提供

init_method：初始化进程组的URL路径，当前支持'tcp://'、'file://'、'env://'，与`store`参数互斥，默认为'env://'

store：进程组共用的K/V键值对存储访问点，与`init_method`参数互斥

timeout：进程组初始化的操作超时时间，默认值为30分钟，对于'NCCL'后端，默认值为10分钟


调用init_process_group的方式主要有两种：

  1. 明确指定`store`，`rank`和`world_size`
  2. 指定`init_method`，它指示在哪里/如何发现对等点。

如果两者都没有指定，init_method则假定为“env://”。因此，`store`和`init_method`是互斥的


使用例子如下：

通过环境变量：

在所有机器上设置以下四个环境变量，所有进程都可以正常连接到master（就是 rank 0 进程）以获取其他进程的信息，并最终与它们建立链接


MASTER_PORT：rank 0 进程的机器上的端口
MASTER_ADDR：rank 0 进程的机器上的 IP 地址
WORLD_SIZE: 进程总数，因此master知道要等待多少worker
RANK: 每个进程的rank，所以进程会知道自己是否是master

通过共享文件系统：

要求所有进程都可以访问共享文件系统，并将通过共享文件协调它们自身。这意味着每个进程都将打开文件，写入其信息，并等待其他每个进程都这样做。之后，所有所需的信息都将可供所有流程使用。为了避免竞争条件，共享文件系统必须支持fcntl锁

dist.init_process_group(
    init_method='file:///mnt/nfs/sharedfile',
    rank=args.rank,
    world_size=4)

通过TCP：

通过提供rank 0进程的IP和端口来实现的，在这里，所有worker都可以连接到rank为 0 的进程并交换有关如何相互联系的信息

dist.init_process_group(
    init_method='tcp://10.1.1.20:23456',
    rank=args.rank,
    world_size=4)

4. Store

Store 是分布式包（distributed package）所提供的分布式键值存储，所有的 workers 都会访问这个存储以共享信息以及初始化分布式包 。用户可以通过显式创建存储来作为init_method的替代。目前有 3 种键值存储：TCPStore， FileStore，和HashStore。



4.1. TCPStore

基于 TCP 的分布式键值存储实现。服务端存储数据，而客户端可以通过 TCP 连接到服务端执行诸如set()插入键值对、get()检索键值对等操作。系统中应该有一个初始化完毕的TCPStore存储服务端，因为客户端将等待这个存储服务以建立连接。函数定义如下：

class TCPStore(Store):
    def __init__(
        self,
        host_name: str,
        port: int,
        world_size: Optional[int] = ...,
        is_master: bool = ...,
        timeout: timedelta = ...,
        wait_for_workers: bool = ...,
        multi_tenant: bool = ...,
        master_listen_fd: Optional[int] = ...,
        use_libuv: Optional[bool] = ...,
    ): ...
    @property
    def host(self) -> str: ...
    @property
    def port(self) -> int: ...

其中重要的参数：

host_name：服务端地址

port：服务端端口

world_size：客户端数量+1，服务端自身也会建立客户端进行链接，默认值为-1

is_master：是否同时初始化服务端，默认值为false

timeout：操作超时时间，默认值为300s

wait_for_workers：是否等待所有客户端与服务端都连接。默认值为true

class FileStore(Store):
    def __init__(self, path: str, numWorkers: int = ...): ...

numWorkers：workers数量，默认值为-1
