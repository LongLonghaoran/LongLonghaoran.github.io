1. 测试环境部署

os:  release V10 (SP2) /(Sword)-x86_64-Build09/20210524

kernel: 4.19.90-80.0.v2307.ky10.x86_64 (使用sp2原生的内核时，在执行rdma link命令时会出现Invalid argument  https://gitee.com/src-openeuler/rdma-core/issues/I6K2FO?from=project-issue)


1、准备两台虚拟机(需要关闭防火墙)，分别安装以下依赖包：

yum install libibverbs-devel libibverbs-utils librdmacm librdmacm-devel librdmacm-utils perl-Switch elfutils-libelf-devel

2、查看当前内核是否支持RXE 

cat /boot/config-$(uname -r) | grep RXE   ###如果 CONFIG_RDMA_RXE 的值为 y 或者 m，表示当前的操作系统可以使用 RXE。

3、加载驱动

modprobe rdma_rxe

4、用户态配置  参考: https://github.com/linux-rdma/rdma-core/blob/master/Documentation/rxe.md

rdma link add rxe_0 type rxe netdev ens3  ##其中 rxe_0 是你希望的 RDMA 的设备名，可任意取名。ens3为 Soft-RoCE 设备所绑定的网络设备名，也就是 ifconfig 中看到的网卡名。

5、查看是否添加成功

  • rdma link

2. 使用perftest测试


perftest是RDMA性能测试工具，该工具由开源社区维护并发布。针对RDMA的5种操作类型：Send、RDMA Read、RDMA Write、RDMA Atomic、Native Ethernet进行带宽和时延测试。


Perftest 是用 Uverbs 编写的测试集合，用于作为性能微基准。它包含一组带宽和延迟基准，例如：

ib_send_lat 	latency test with send transactions
ib_send_bw 	bandwidth test with send transactions
ib_write_lat 	latency test with RDMA write transactions
ib_write_bw 	bandwidth test with RDMA write transactions
ib_read_lat 	latency test with RDMA read transactions
ib_read_bw 	bandwidth test with RDMA read transactions
ib_atomic_lat	latency test with atomic transactions
ib_atomic_bw 	bandwidth test with atomic transactions

Raw Ethernet interface benchmarks:
raw_ethernet_send_lat  latency test over raw Ethernet interface
raw_ethernet_send_bw   bandwidth test over raw Ethernet interface


注：perftest 软件包需要使用cuda进行编译，才能使用 GPUDirect 功能。参考：https://docs.oracle.com/zh-cn/learn/gpudirect-rdma-ib-write-bw/index.html#acknowledgment


安装使用：

可测试的架构:  i686, x86_64, ia64

1.以rpm包安装

  • yum install perftest （koji上有该rpm包）

2.源码编译安装

-cd perftest/

-./autogen.sh

-./configure 

-make

-make install
