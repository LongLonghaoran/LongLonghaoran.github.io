笔记本：容器镜像
从高层级来看，镜像清单包含关于镜像内容及其依赖项的元数据，其中包括一个或多个文件系统层变更集归档的内容可寻址标识，这些归档文件解压后将构成最终可运行的文件系统。镜像配置则包含诸如应用程序参数、环境变量等信息。镜像索引是更高层级的清单，它指向一组清单和描述符。通常这些清单可能提供镜像的不同实现方式，可能因平台或其他属性而异。

OCI Image Media Types:
OCI 镜像媒体类型 (mediaType) 是 OCI 镜像规范中的核心组成部分，其主要作用体现在以下几个方面：
‌标识内容格式与语义‌：
mediaType 明确标注了某个文件或数据流的具体格式类型（如 JSON 配置文件、压缩的 tar 包层）以及遵循的规范版本（如 v1）。例如：
application/vnd.oci.image.manifest.v1+json 表明文件是一个符合 OCI 镜像规范 v1 版本的清单 (Manifest) 文件，格式为 JSON。
application/vnd.oci.image.config.v1+json 表明文件是一个符合 OCI 镜像规范 v1 版本的配置 (Config) 文件，格式为 JSON。
application/vnd.oci.image.layer.v1.tar+gzip 表明数据是一个符合 OCI 镜像规范 v1 版本的层 (Layer)，其格式是经过 gzip 压缩的 tar 包。35
这确保了接收方能准确理解如何处理和解析该内容。
‌确保兼容性与互操作性‌：
通过标准化的 mediaType，不同的容器运行时（如 containerd, cri-o, Docker Engine）、镜像构建工具（如 Buildah, Kaniko）和镜像仓库（如 Docker Hub, Harbor, Amazon ECR）能够基于一致的标识符理解镜像组件的结构和含义。14
这是实现容器技术“一次构建，随处运行”愿景的关键技术基础之一。无论底层使用何种具体的容器软件，只要它们遵循 OCI 规范并识别这些 mediaType，就能正确无误地处理同一个镜像。14
OCI Image manifest specification:
镜像清单规范主要有三大目标。其一是实现内容可寻址镜像，通过支持一种镜像模型，将镜像配置进行哈希处理，从而为镜像及其组件生成唯一标识符。其二是支持多架构镜像，借助引用各平台特定版本镜像清单的"聚合清单"（OCI中体现为镜像索引）实现。其三是确保规范可转换为OCI运行时规范。
