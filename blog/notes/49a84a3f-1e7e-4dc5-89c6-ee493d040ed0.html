笔记本：Kubernetes学习笔记
    kubectl cluster info之后返回访问的url，浏览器打开url后提示是https连接，继续前往之后，提示
"message": "forbidden: User \"system:anonymous\" cannot get path \"/\"",
这是由于访问url没有登陆导致？看了kubectl get secret -A好像跟service-account-token 有关系

通过expose将deployment端口暴露到外部:
kubectl expose deployment hello-node --type=LoadBalancer --port=8080
通过minikube service将service暴露给外部
通过kubectl proxy命令可以启动代理来访问k8s集群中的应用通过api端点 /api/v1/namespaces/:namespace/pods/:pod_name/可以访问

通过kubectl create deployment deployment-name --image=xxx
通过指定的镜像来创建部署

通过kubectl service或者proxy来暴露应用以供外部访问

通过kubectl scale命令来扩缩应用: kubectl scale deployment k8s-bootcamp --replicas=4

对应用进行滚动更新和回滚:
直接使用set子命令就可以 kubectl set image deployments deployment-name container-name=image-name
完成后可以通过 kubectl rollout status deployment deployment-name来检查滚动更新状态
