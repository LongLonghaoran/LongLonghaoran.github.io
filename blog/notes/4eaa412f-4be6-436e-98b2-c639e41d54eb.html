0.原型的指向
    构造函数中的this就是实例对象
    原型对象方法中的this也是实例对象

    function Person(age){
        this.age = age;
    }
    Person.prototype.eat = function(){
        console.log(this);
        console.log("Person吃东西");
    }
    function Student(){
    }
    //更改Student构造函数的原型对象
    Student.prototype = new Person();
    var stu = new Student();
    stu.eat();

    实例对象和原型对象之间的联系也叫作“原型链”

    其实原型对象也是个对象，所以原型对象中也有一个__proto__属性，它又指向另一个构造函数的prototype,比如Object
    所有对象的原型链上的最终指向都是Object
    而Object.__proto__的指向是null

1.继承
    面向对象的语言是为了多态服务的
    1.
        所谓继承就是通过更改原型的指向，将构造函数的prototype指向它需要继承的 父 构造函数的实例对象即可。
        将需要继承的属性写在父构造函数中，实例对象在调用属性 方法时会从自身开始查找，然后依次找
        原型对象 原型对象的原型。。
        比如,狗作为父构造函数 哈士奇作为子构造函数:
            function Dog(name){
                this.name = name;
            }
            Dog.prototype.say = function(){
                console.log("汪汪");
            }
            function Haski(sex){
                this.sex = sex;
            }
            Haski.prototype = new Dog('狗子');
            Haski.prototype.play = function(){
                console.log('我的娱乐活动就是拆家');
            }
            var haski = new Haski('雄性');
            haski.play();
            haski.say();
    2.借用构造函数继承:
        上面虽然实现了继承，但是有一个问题，更改原型指向时，直接将原型指向了父构造函数
        的实例对象，这样会导致，之后所有子构造函数创建的实例对象在他们的公有属性 或者说
        继承的属性的值，全部都是一样的，而且在改变原型指向我们需要实例化父构造函数，这
        个操作不应该发生在对象定义阶段，因为按理来说这个时候实例对象中任何属性的值，我们
        都不知道。

        解决方法,直接调用父级的构造函数:
            function Person(name, age){
                this.name = name;
                this.age = age;
            }
            function Student(name, age, sex){
                Person.call(this, name, age)
                this.sex = sex;
            }
            Student.prototype = new Person();
            var stu = new Student('媛媛', 16, '女');
        call方法:
            调用父级构造函数，而且将父级构造函数中的this替换为作为参数的this
            类似的可以用bind方法也能实现
            Person.bind(this)(name, age)
            bind方法也是更改方法内的this指向
        注意:
            通过这种方式可以让子级构造函数创建的实例对象的属性不一致，但是
            实际上是每个子级构造函数都拥有了子级的和父级构造函数同名的属性,
            创建子级实例对象之后，它的原型对象中(也就是父级的实例对象)的同名属性
            的值都是undefined

    3.拷贝继承
        1.用for in遍历对象的每个键值对,将其赋给新的的对象
            obj1 = new Student();//obj1是个实例对象
            for( var key in obj1){
                obj2[key] = obj1[key];
            }
            这种拷贝也叫 浅拷贝，只能拷贝对象自身的属性，而不能拷贝它继承的那些属性,
            因为__proto__这个属性并不会被比遍历函数访问到,说白要是循环能够取到__proto__
            原型属性，那也是可以深拷贝的

2.函数的不同方式
    函数定义的三种方式:
        1.函数声明,也叫 普通的命名函数的声明方式
            function foo(){

            }
        2.函数表达式(匿名函数)
            var foo = function(){}
        3.new Function
    一个特殊的情况:
            if(true){
                function f(){console.log('结果为真')};
            }else{
                function f(){console.log('结果为假')};
            }
            f();
        这段代码如果在ie11以下的Ie浏览器中结果是第二个function, 结果为假。
        这是一个问题，所以为了所有浏览器都正常使用，当我们在if else等条件判断中定义函数的
        时候，我们应该使用条件表达式:
            var f;
            if(true){
                f =function (){console.log('结果为真')};
            }else{
                f= function (){console.log('结果为假')};
            }
            f();

3.this指向
    记住一个原则即可:
        所有函数中的this指向都是指调用该函数的实例对象
    举几个例子:
        1.普通函数
            function f(){console.log(this)};
            f();
            这相当于是window.f()
            window = {
                f: function,
                ...
            }
        2. 对象.方法,这就是最原始的方法的调用形式,所有的方法调用本质上都和它是一样的原理
            var obj = {
                f: function(){console.log(this)}
            }
            obj.f();
        3.定时器方法
            setInterval(function(){
                console.log(this);
            }, 1000);
            本质上是window的一个方法属性,所以this是window
        4.构造函数中的this:
            function f(){
                console.log(this);
            }
            new f();

5.函数也是对象
    函数也是对象，但对象不一定是函数
    函数拥有prototype属性，这是一般对象所没有的，而且函数作为对象，也拥有__proto__属性
    所有的函数实际上都是Function构造函数(大写的F)创建出来的实例对象。
    可以这样使用:
        var f = Function('n', 'console.log(n)');
        f();
