Operator和kubebuilder中都提到Operator是基于kubebuilder的plugins实现的

在kubebuilder中,版本3.13.0默认plugins含有go/v4
在operator-sdk中,版本v1.31.0-2-g8f24aa56默认含有go.kubebuilder.io/v3, 此外go/v4在这里的版本号仍然处于go.kubebuilder.io/v4-alpha
Plugins:
在macOS Apple silicon中直到go.kubebuilder.io/v4-alpha才获得支持

扩展kubebuilder的两种方式：
复用现有插件，将kubebuilder作为一个库引入，构建一个你自己的kubebuilder
Extending the CLI and Scaffolds扩展CLI以及脚手架
通过扩展kubebuilder来让你的项目有相同的cli特性并提供插件脚手架

扩展cli的具体实现可以参考operator-sdk的实现代码，它将kubebuilder作为工具库引入到
自己的项目中，并通过kubebuilder提供的cli包来扩充plugins选项

package main

import (
// Import all Kubernetes client auth plugins (e.g. Azure, GCP, OIDC, etc.)
// to ensure that `exec-entrypoint` and `run` can make use of them.
_ "k8s.io/client-go/plugin/pkg/client/auth"
"sigs.k8s.io/kubebuilder/v3/pkg/plugin"
"sigs.k8s.io/kubebuilder/v3/pkg/plugins/golang"

log "github.com/sirupsen/logrus"
golangv2 "sigs.k8s.io/kubebuilder/v3/pkg/plugins/golang/v2"

"sigs.k8s.io/kubebuilder/v3/pkg/cli"
)

func main() {
c := GetPluginsCLIAndRoot()
c.Run()
}

func GetPluginsCLIAndRoot() *cli.CLI {
gov2Bundle, _ := plugin.NewBundle(
golang.DefaultNameQualifier,
golangv2.Plugin{}.Version(),
golangv2.Plugin{},
)
c, err := cli.New(
cli.WithCommandName("my-bin-builder"),
cli.WithVersion("版本号？"),
cli.WithPlugins(
gov2Bundle, // Deprecated
),
cli.WithCompletion(),
)
if err != nil {
log.Fatal(err)
}

return c
}


创建外部的独立的插件：
作为一个独立的二进制运行，kubebuilder直接调用该二进制来与之交互，因为其作为独立程序运行，所以可以使用任何编程语言来实现。
现有的例子有helm-operator和ansible-operator
实现方式：
原理：kubebuilder和插件之间通过stdio来通信，所以开发的插件一定不要有任何期望之外的输出到标准输出，如果需要打印日志之类的，应该考虑将日志输出到文件中去
