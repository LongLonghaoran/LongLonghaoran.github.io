1.原型链:
    绝大多数对象最终都会继承自Object.prototype,注意：Object.create(null)构造出的对象就没有原型,即便通过obj.__proto__再次赋值对象都不行

2. 方法的重写：如果原型中存在一个方法，那么在继承这个原型的一个子原型中可以写一个方法名相同的方法，叫做方法重写。
        Person.prototype.toString = function(){
            console.log("呵呵，这是我重写的方法");
        }
        function Person(){}
        function People(){}
        //会显示我自定义的方法内容
        new Person().toString();
        //会显示Object的方法
        new People().toString();

        与之类似的就是js自带的几个类的方法重写: Number String Boolean 等等

3.js中数值计算会有误差，做运算时，正整数最多可以计算16位，小数最多可以计算到小数点后16位


4.call/apply,
    * call才是方法调用的真实面目,使用方法为： 方法名.call
    new Number(123).toString.call(), call方法的独特用法在于，call方法的第一个参数可以传入其他其他参数，此时，调用的方法中所有this指向都更改到传入的这个对象上来，比如
        function Person(name, age){
            this.name = name;
            this.age = age;
        }

        obj = {}
        Person.call(obj, 'longhr', '18')

    一个经典的例子:
        有两个构造函数，其中有很多属性初始化都是重复的，希望可以简写代码让代码更少:
            function Person(name, age, height){
                this.name = name;
                this.age = age;
                this.height = height;
            }

            function Student(name, age, height, grade){
                this.name = name;
                this.age = age;
                this.height = height;
                this.grade = grade;
            }

            //Student方法就可以通过call进行改写
            function Student(name, age, height, grade){

                Person.call(this, name, age, height);
                this.grade = grade;
            }
    * apply,它的功能和call一致，主要在调用形式上有区别:
        call 需要把实参按照形参的个数传进去，apply的实参必须传一个arguments数组,
        比如将上面的call改为 Person.apply(this, [name, age, height])

习题: js的 call和apply的区别时什么，作用是什么？
    作用: 改变this指向
    区别: 传参列表不同

5. 继承
  1.传统形式 -> 原型链,通过prototype = new Father()来继承
    过多的继承了没用的属性s
  2.借用的构造函数,即apply/call方式
    不能继承借用构造函数的原型
    每次构造函数都要多走一个函数
  3.共享原型,直接将想要继承的原型直接赋给目标对象
    不能随便改动自己的原型
    Father.prototype.lastName = 'Long'
    function Father(){

    }
    Son.prototype = Father.prototype
    function Son(){

    }

    两个prototype是一样的
    new Father -> {
      __proto__: {a: 1}
    }
    new Son -> {
      __proto__: {a: 1}
    }

    但是缺点就是多个类的原型是指向的同一个，更改一个的原型，就会影响到其它的
    构造函数/对象
  4.圣杯模式
    通过一个中间的构造函数:
      function F(){}
      F.prototype = Father.prototype
      Son.prototype = new F()

      new Father()->{
        __proto__: {a: 1}
      }

      new F()->{
        __proto__: {a: 1}
      }

      new Son() ->{
        __proto__: {
          __proto__: {a: 1}
        }
      }

      如此一来，即便这样 Son.prototype.b = 1;也不会影响到Father类了

6.Javascript中构建私有属性:
    function Test(num){
        var a = num;
        this.getA = function (){a};
    }

    test = new Test();
    执行完new Test()之后，test函数以及上下文执行期的引用在外部就拿不到了，通过test.a 或者直接调用a都是访问不到的，但是test.getA()是可以拿到的，
    因为getA将函数的作用域保存下来了
