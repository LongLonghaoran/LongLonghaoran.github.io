笔记本：Kubernetes学习笔记
Pod是k8s调度的最小原子单位
Pod里的容器具有 “超亲密关系”：
    所谓超亲密关系，指的是容器之间可以直接使用localhost或者直接使用socket文件进行通信
要解决的问题：
    如何让一个Pod里的多个容器之间最高效的共享某些资源？
[图片/附件]

共享网络：
    通过infra container实现，其实质是启动一个额外的小型容器
共享存储：
    通过pod层级的volume实现，这样各个容器就都可以共享这个volume
设计模式：
    重点关注sidecar:
        通过在pod中定义专用容器，来执行主页午容器需要的辅助工作:
        1.日志处理容器，通过一个共享volume来访问应用日志，并且定义一个专用容器来处理这些日志文件
        2.代理容器，比如应用需要与许多的第三方api进行通信，为此可能需要建立一个配置文件来专门维护这些第三方的ip地址，但是通过专用容器，我们可以将这个工作转移到sidecar中，然后应用容器直接访问sidecar就好了，因为容器之间访问都是可以通过localhost进行访问的，所以应用容器只需要像访问本地应用一样访问调用接口就可以了，这也算是一种对应用的解耦

    故障排除:
        最常见的操作可以使用以下 kubectl 命令完成：
  • kubectl get - 列出资源
  • kubectl describe - 显示有关资源的详细信息
  • kubectl logs - 打印 pod 和其中容器的日志
  • kubectl exec - 在 pod 中的容器上执行命令
      
    访问应用:
        可以通过kubectl proxy启动一个代理来访问到节点,kubectl proxy启动的代理默认会启动一个http服务，监听8001端口，可以通过它查看各个pod的信息，也可以通过其代理来访问Pod内应用
        比如： http://127.0.0.1:8001/api/v1/namespaces/default/pods/nginx-deployment-5b9cf4fbdd-r2gg2/proxy/
        可以通过kubectl exec 在容器中执行命令就和docker exec 类似通过kubectl exec -it $POD_NAME -- bash
     使用service暴露应用:
        由于Pode是有生命周期的，它会消亡和替换，ip地址也会变更，Service定义了Pod逻辑集和访问Pod的协议。
  • ClusterIP (默认) - 在集群的内部 IP 上公开 Service 。这种类型使得 Service 只能从集群内访问。
  • NodePort - 使用 NAT 在集群中每个选定 Node 的相同端口上公开 Service 。使用<NodeIP>:<NodePort> 从集群外部访问 Service。是 ClusterIP 的超集。
  • LoadBalancer - 在当前云中创建一个外部负载均衡器(如果支持的话)，并为 Service 分配一个固定的外部IP。是 NodePort 的超集。
  • ExternalName - 通过返回带有该名称的 CNAME 记录，使用任意名称(由 spec 中的externalName指定)公开 Service。不使用代理。这种类型需要kube-dns的v1.7或更高版本。
         Kubernetes 的 Service 是一个抽象层，它定义了一组 Pod 的逻辑集，并为这些 Pod 支持外部流量暴露、负载平衡和服务发现。

            
    通过kubectl expose 来暴露应用，然后通过minikube ip+port就可以访问到应用了
        至于kubectl expose命令，它可以用于暴露pod deployment等

    可以通过Kubectl label pods pod_name key=value 来给Pod打上Label，之后可以通过该label来筛选pod，比如
        kubectl get pods -l key=value,同样的操作可以应用在deployment service上面

扩缩应用程序:
  扩缩 是通过改变 Deployment 中的副本数量来实现的。
    运行kubectl run 命令的时候可以通过replicas=n来指定副本数量
    通过kubectl scale deployments/kubernetes-bootcamp --replicas=4可以扩容指定的deployment
    replicas可增可减
更新应用程序:
    k8s支持滚动更新，也就是允许新的实例逐步更新Pod实例，零停机进行Deployment
    可以通过kubectl set image deployments $deployment_name来更新deployment中的镜像
