笔记本：前端学习
html5:
    1.h5并不是新语言，而是html语言的第五次重大修改之一的版本
        html:5 h5页面
        html:xt h4页面
        html:4s h4的严格模式

    2.h5有一些新增的语义标签，所谓语义标签，就是语义性较强的标签，一看名字
        就能够知道它的作用:
            header
            footer
            main
            article
            aside
            nav
            但是这些标签实际的作用和div是一样的,只是语义性更加明确,使用起来和div没有
            本质的区别
        新的语义化标签的兼容性问题:
            虽然说ie9以及上支持h5标签,但是会将其默认转换为行级元素,解决方案:
                1.给元素增加css: display: block
                    aisde,main,article {
                        display: block;
                    }
            在IE8中完全不支持语义标签,解决方案:
                1.手动创建标签,并且指定display: block:
                    <style>
                        header,... {
                            display: block;
                        }
                    </style>
                    <script>
                        document.createElement("header");
                        document.createElement("nav");
                        document.createElement("main");
                        document.createElement("article");
                        document.createElement("aside");
                        document.createElement("footer");
                    </script>
                2.引入第三方插件:
                    <script src="../js/html5shiv.min.js"></script>

    3.新增的属性:
        autocomplete="on/off"是否自动填充
            启用有两个条件:
                1.必须成功提交过
                2.必须有name属性
        required: 必填
        pattern: 输入框的正则验证:
            使用场景: h5的input已经新增了type="tel",但是它实际并没有限制输入的类型，
            它的作用只是让输入框在移动端使用时可以弹出一个数字输入框，所以这个时候
            需要我们自己来写验证，就可以用到pattern:

            例子:   <input type="tel" required pattern="^(/+86)?1\d{10}$">
        form:
            这个属性可以让在form表单之外的输入框的值，在表单提交时它的值也跟着一起提交
            form="表单id值"
        datalist:
            虽然效果不错，但是，兼容性很差，比如firefox中就不支持datalist中的option的value
            属性，所以一般开发中是几乎不适用它的。
    4.新增的事件:
            oninput: 可以实时动态的监听input框的变化，一旦内容改变就会触发
            onkeyup: 实时的监听键盘，键盘弹起就触发
            onkeydown: 同onkeyup,效果相反
            oninvalid: 当验证不通过时触发
        表单的默认提示是可以更改的,通过配合oninvalid来监听一个输入框验证不通过的事件。
            使用setCustomValidity来实现
            document.getElementById("userPhone").oninvalid=function(){
                this.setCustomValidity("请输入合法的11位手机号");
            }

    6.h5的音频和视频:
        使用多媒体有4种方式:
            1.embed,直接插入视频文件，它的本质是调用本机已安装的软件，有兼容性
            2.flash，需要安装flash,学习flash，增加了成本,苹果设备不支持flash
            3.audio标签
            4.video标签
                注意: 一般情况下只会设置高度或者宽度，除非你设置的宽高刚好是
                等比的

    7.h5操作元素类样式:
        1.通过querySelector以css选择器的方式获取元素，一次只能获取匹配的第一个元素，
        如果需要获取所有元素需要使用querySelectorAll,如果是通过querySelectorAll
        获取的元素，也可以在此基础上使用[下标]来继续获取具体元素
        2.classList方法，querySelector配合classList以及 add remove方法可以
        给元素增加、删除class
            document.querySelector("li").classList.add("red")
            toggle:
                document.querySelector("li").classList.toggle("red")
                如果元素有red则移除red,如果没有则添加red
            contains:判断元素是否包含指定class类名
    8.h5自定义属性:
        定义:
            1.data开头
            2.data-后必须至少有一个字符
            3.多个单词之间使用 - （横杠）连接
        建议:
            1.名称都应该使用小写，不要包含任何大写字符
            2.名称中要有任何的特殊符号
            3.名称不要使用纯数字
        <p data-school-name="itcast">传智播客</p>
        使用:
            通过dataset方法，并且配合驼峰式命名来获取:
                document.querySelector("p").dataset['schoolName']
