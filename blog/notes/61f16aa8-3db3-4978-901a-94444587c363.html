1.通过配置文件来配置webpack
    一个最基本的需求，我们不希望每次都要显示指定输入和输出的文件路径，我们希望直接执行Webpack命令就会自动去执行
    这个时候就可以通过webpack.config.js文件去配置
// 这个配置文件其实就是一个js文件,通过node中的模块操作，向外暴露了一个配置对象


// 由于webpack是基于node.js构建的，所以可以使用nodejs中的核心模块
const path = require('path');
module.exports = {
    entry: path.join(__dirname, './src/main.js'),
    output: {
        path: path.join(__dirname, './dist'),
        filename: 'bundle.js'
    }
}

2.使用webpack-dev-server这个工具实现自动打包，因为在开发过程中不断的自己敲命令打包也是很麻烦的,我们需要
    可以自动打包
    0.webpack-dev-server要求在在项目本地中需要安装webpack，注意是在本地安装，即便已经安装了全局的webpack也不行 
    1.安装 npm i webpack-dev-server -D
            这是安装在项目目录下
    2.使用
        由于我们是在项目中本地安装的,所以无法把它当做脚本命令，只有安装在全局的工具才能在终端中正常执行
        在package.json中配置script属性，这是定义命令的属性，
        
  "scripts": {
    "dev": "webpack-dev-server"
  },
    3.执行 npm run dev
        会启动一个服务监听项目目录，其中打包好的文件可以通过服务的根路径直接访问到
    4.webpack-dev-server帮我打包生成的bundle.js文件，并费油存放到实际的物理磁盘上，而是托管到电脑的内存中
        所以在项目根目录是找不到这个bundle.js的        
3.webpack-dev-server常用命令参数
    1. --open 启动服务后自动以默认浏览器打开
    2. --port 3000 以3000端口运行
    3.--contentBase src 以src为根路径
    4.--hot 热重载/热更新 
        每次我们 ctrl+s 保存代码之后，webpack通过补丁的方式更新打包后的文件，而不是完全重新生成一份文件

4.webpack-dev-server命令参数的第二种配置方式，通过webpack.config.js来配置
    稍微复杂点的部分就是hot的开启的配置:
// 这个配置文件其实就是一个js文件,通过node中的模块操作，向外暴露了一个配置对象


// 由于webpack是基于node.js构建的，所以可以使用nodejs中的核心模块
const path = require('path');
// 启用热更新
const webpack = require('webpack');
module.exports = {
    entry: path.join(__dirname, './src/main.js'),
    output: {
        path: path.join(__dirname, './dist'),
        filename: 'bundle.js'
    },
    devServer: {
        // 这是配置dev-server命令参数的第二种形式
        open: true,
        port: 3000,
        contentBase: 'src',
        hot: true
    },
    plugins: [
        // 这是一个热更新的模块的对象
        new webpack.HotModuleReplacementPlugin()
    ]
}

5.html-webpack-plugin插件   --------------生成内存里的首页
    1.安装 npm i html-webpack-plugin
    2.webpack.config.js配置
        在上面第4步基础上，先引入该插件，然后在plugin下追加这个插件的实例化对象
const htmlWebpackPlugin = require('html-webpack-plugin');

        new htmlWebpackPlugin({
            template: path.join(__dirname, 'src', 'index.html'),
            // 指定生成的页面名称
            filename: 'index.html'
        })

    3.它的作用，使用了它之后，我们都不用手动在页面中通过script引入我们的main.js，这个插件会在生成的页面
        中自动在页面底部加上script src='/bundle.js'

6,配置预处理css样式表的第三方loader
    1.在src/css下新建一个 index.css文件
    2.这次我们不再在页面上通过link标签导入，而是从main.js中导入
        import './src/index.css'
        但是webpack默认只能打包 js类型的文件,所以需要一个loader
    3.需要2个loader: style-loader css-loader
        同样也是安装在开发环境依赖下:
        npm i style-load css-loader -D
    4.webpack.config.js新增一个配置节点
        module: {}， 它是一个对象，有个rules属性，是个数组，它存放了所有第三方文件的匹配和处理规则
            每个规则用一个对象来描述
    // 这个节点用于配置所有第三方模块加载器
    module: {
        rules: [
            // 它的原理是用正则表达式去匹配对应的文件类型（文件后缀）
            {
                // 匹配所有以 .css结尾的文件
                test: /\.css$/,
                //匹配上的文件就使用以下的模块加载器来处理
                // webpack处理第三方文件类型时，会按照以下数组顺序中 “从右至左” 的顺序，依次调用加载器来
                // 进行处理,当最后一个loader处理完毕，会把处理结果直接交给webpack打包合并,并最终输出到
                // bundle.js中去
                use: ['style-loader', 'css-loader']
            }
        ]
    }

7.有了上面的方案，我们还可以继续安装 less-loader，然后配置来支持.less文件
    需要注意的是，配置加载器的时候，less处理完之后还只是css,所以还是需要加上css-loader 和 style-loader

8.sass也是一样的道理
    需要安装 sass-loader
