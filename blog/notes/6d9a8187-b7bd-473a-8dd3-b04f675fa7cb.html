1.事件:
    1.绑定事件的区别
        1.this不同
            addEventListener中表示当前绑定事件的对象
            attachEvent中表示window
        2.参数不同
        3.事件名称: addEventListener不带on, attachEvent带on
    2.移除绑定的方式以及区别和兼容代码
        1.比如通过onclick = function(){}绑定的事件，只需要再次将onclick = null即可解绑事件
        2.通过addEventListener("click", function(){})绑定的事件，通过
            removeEventListener(type, handler, false)
            type是事件名称,handler是函数名称,也就是说之前绑定事件的时候，就通过
            命名函数的名称还是绑定事件，解绑事件的时候将函数名称作为参数，即可解绑
            这个事件:
                function f(){
                    console.log("111");
                }
                my$('btn').addEventListener("click", f);
                my$('btn2').addEventListener("click", function(){
                    my$('btn').removeEventListener("click", f);
                });
        3.attachEvent
            通过detachEvent
                function f(){
                    console.log("111");
                }
                my$('btn').attachEvent("onclick", f);
                my$('btn').attachEvent("onclick", function(){
                    console.log("2");
                });
                my$('btn2').onclick = function(){
                    my$("btn").detachEvent('onclick', f);
                }
        4.解绑事件的兼容代码:
            上面的代码看上去复杂，如果封装成一个函数来执行的话，就方便很多:
            function addEventListener(element, type, fn){
                if(element.addEventListener){
                    element.addEventListener(type, fn, false);
                }else if(element.attachEvent){
                    element.attachEvent("on" + type, fn);
                }else {
                    element.onclick = fn;
                }
            }
            function removeEventListener(element, type, fn){
                if(element.addEventListener){
                    element.removeEventListener(type, fn, false);
                }else if(element.attachEvent){
                    element.detachEvent("on" + type, fn);
                }else {
                    element.onclick = null;
                }
            }
    3.事件的三个阶段
        1.事件捕获阶段
        2.事件目标阶段
        3.事件冒泡阶段
        addEventListener的第三个参数，true/false,表示事件是捕获(从外向里) 还是 冒泡(从里向外)
        在事件对象参数中有一个属性 e.eventPhase表示事件是处于事件的哪个阶段

        比如有如下结构:
            <div id='dv1' class="d1">
                <div id="dv2">
                    <div id="dv3"></div>
                </div>
            </div>
        并且着三个div都注册了相同的点击事件
            var objs = [my$("dv1"), my$("dv2"), my$("dv3")];
            objs.forEach(function(e){
                e.addEventListener("click", function(e){
                    console.log(this.id + "-----" + e.eventPhase);
                }, false)event
            });
        点击最里面的dv3的话，外层的dv1 dv2的相同事件的事件函数也会触发，而且这个顺序是
        dv3 > dv2 > dv1，这个就叫做"事件冒泡",另外，addEventListener的第三个参数表示
        事件是否是按照捕获顺序来，true则表示按照捕获来，false则表示按照默认的事件冒泡
        即从里向外的顺序来，如果是true的话，那么事件的执行顺序就是 dv1 > dv2 >dv3了,

        至于事件的三个阶段，这个表示每个元素在事件触发时，它们所处在事件的哪个阶段
        2---目标阶段，即表示 真正触发事件的那个元素
        1---捕获阶段，当addEventListener第三个参数为 true,那么触发事件的元素的外层
        元素所处的就是事件捕获阶段
        3---冒泡阶段，当第三个参数为false,那么触发事件的元素的外层元素所处的就是事件
        冒泡阶段

    4.事件冒泡
        概念: 多个元素嵌套，有层次关系,这些元素都注册了相同的事件,如果子元素的事件触发了，
        那么父级以及更上级的父元素的事件也会触发,这样不符合我们的预期

        阻止事件冒泡 ：
            前言：
                事实上每个事件回调函数在被调用的时候，都会有一个参数传进来,这个参数是
                NodeEvent 事件处理对象,当然如果我们不用这个参数我们可以不写，在js中这是
                允许的，因为：方法的形参可以省略，我们甚至可以通过arguments来一次性获取所有
                参数,另外这也给我一个启示，有些系统自带的回调函数，如果我不清楚它的参数
                有哪些，那么我可以在方法内打印arguments来查看
            阻止事件冒泡方法:
                1.IE8和chrome中:
                    在需要阻止的事件函数中
                        添加window.event.cancelBubble=true;
                2.在chrome和firefox中:
                    e.stopPropagation();
                    arguments[0].stopPropagation();
                    上面的e和arguments[0]是同一个意思,e就表示直接在方法参数中写上e
                    这个方法不适用于IE8，原因是IE8不支持回调函数的event参数
                3.事实上e == window.event, window.event和回调函数传过来的事件对象是同一个对象
    5.为同一个元素绑定多个不同的事件,指向的是同一个事件处理函数
        情况如下:
            function f1(e){ console.log(e)};
            my$("btn").onclick = f1
            my$("btn").onmouseover = f1
            my$("btn").onmouseout = f1
            给一个元素绑定多个事件,事件处理函数都是同一个
        观察事件对象参数 e,其实会发现,e里面有相当多的预置的属性，这里我们取其中一个来讲解
        type属性,即 e.type，它表示的是事件的名称，有 click mouseover ..等等，总之就是事件名称的标准写法(没有on,加了on其实是微软的写法)
        也就意味着，即便我们只绑定一个回调函数，但是可以通过type判断，进而在一个函数中实现对不同事件
        进行不同操作

        <div id="box">

        </div>

            var keyWords = ["小杨才是最纯洁的", "小杨才是最帅的", "小段是最x猥琐的", "小超是最龌龊的", "传智播客是一个培训机构", "传说在传智有个很帅很纯洁的小杨", "苹果好吃", "苹果此次召回还是没有中国"];
            my$('text').addEventListener('keyup', function(){
                var text = this.value;
                result = [];
                keyWords.forEach(function(e){
                    if(e.indexOf(text)!=-1){
                        result.push(e);
                    }
                });
                if(this.value == '' || result.length==0){
                    my$('pop').innerHTML = "";
                    return;
                }
                my$('pop').innerHTML = "";
                var dvObj = document.createElement('div');
                my$('pop').appendChild(dvObj);
                dvObj.id = 'dv';
                dvObj.style.width = '350px';
                dvObj.style.border = '1px solid grey';
                result.forEach(function(e){
                    var p = document.createElement('p');
                    p.innerText = e;
                    dvObj.appendChild(p);
                    p.addEventListener("click", moveSelected, false);
                });
            });
            function moveSelected(e){
                my$('text').value = e.target.innerText;
                my$('pop').innerHTML = '';
            }
        </script>

4.有几个方法，能不用尽量不用
    alert prompt confirm（确认对话框），因为不同浏览器上外观都不相同，而且对话框会阻塞浏览器
    一般都自己来写
5.页面加载的事件:
    由于html顺序执行代码的原因，所以如果将事件绑定代码写在被绑定的元素之上（比如将js写在页面顶端）
    那么绑定事件的时候，页面还没加载完毕，这样就会因为找不到元素而报错。
    所以我们需要将js写在页面低端 或者 等到页面加载完毕再执行js
    window.onload = function(){
        //只要页面加载完毕，这个事件就会触发
        页面中的所有内容，标签，属性，文本
    }
    window还是可以省略:
    onload = function(){
            //只要页面加载完毕，这个事件就会触发
            页面中的所有内容，标签，属性，文本
    }
    当然，写代码时还是要尽可能地将代码写在页面底端

8.navigator对象
    navigator.platform 判断浏览器所在系统平台类型
    navigator.userAgent 可以判断用户浏览器的类型
    但是，mdn上这么解释：

        PS: 建议使用特征检测来检测web 新特性的支持情况，从而实现对浏览器UX的渐进增强的优化！

        通过检测 userAgent 的值来判断浏览器类型是不可靠的，也是不推荐的，因为用户可以修改userAgent 的值。比如：

        Firefox 中，你可以通过在 about:config 页面添加并修改general.useragent.override 选项的值来覆盖默认的用户代理字符串。一些Firefox扩展也可以用其他方式修改这个值，但它们通常只会修改发送到服务器的User-Agent请求头的值，而不会影响 JavaScript 代码中获取到的 window.navigator.userAgent 属性的值。
        Opera 6+ 中允许用户通过菜单选择不同的用户代理字符串。
        Microsoft Internet Explorer 使用注册表来配置自己的用户代理字符串。
        Safari 和 iCab 也允许用户修改其用户代理字符串，来将自己伪装成 Internet Explorer 或 Netscape。
