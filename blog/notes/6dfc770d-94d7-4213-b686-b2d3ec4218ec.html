笔记本：前端学习
1.require方法的加载规则:
    1.三种模块的加载    
        1.核心模块
            模块名
        2.用户自定义模块
            用户自己存放的模块所在路径
        3.第三方模块
            模块名
    2.require会优先从缓存加载，而且加载过的模块，不会再次加载
    
    3.判断模块标识,也就是require的参数
        1.路径形式的模块:
            以 ./ ../ /xxx 开头的参数名称, 则按照 相对路径 或者 绝对路径来查找
        2.如果是非路径形式的模块标识,则node将其视为 核心模块 或者 第三方来加载
            1.核心模块
                核心模块文件已经被编译进入二进制文件，我们只需要按照名字来加载即可
            2.第三方模块
                1.所有第三方模块都必须通过npm来下载，使用的时候就可以通过require('包名')的方式
                    来进行加载才可以使用

                2.不可能有任何一个第三方包和核心模块的名字是一样的

                3.第三方模块的加载规则
                    加载顺序，以 art-template为例，我们先npm install art-template安装该模块
                        1.先找当前目录中的 node_modules目录中，查找是否有指定名称的目录,
                        2.再查找该目录的下的package.json文件，
                        3.再找其中的main属性
                        4.main属性就记录该模块的入口模块,比如 main: './index.js'， 即表示这个模块入口是index.js
                            中，其实也可以相当于是 require了 index.js
                        5.如果package.json不存在或者 main属性指定的文件不存在 或者 main属性不存在，那么默认会去加载模块陌路中                                的index.js
                        6.额外情况:
                            上面5点规则如果都不满足的话，那么继续进入上一级目录的中的node_modules目录查
                            找，还是找不到的话，继续往上一级，一直会找到 根目录下的 node_modules，如果还找
                            不到，则提示 找不到该模块，加载失败
                        
                    
    4.关于 package.json 包描述文件
        1.dependencies
            首先这个文件保存了当前项目的插件依赖的信息,一般它里面会有一个 dependencies属性，就是用于
            记录该项目依赖的其他包
            npm install 的时候，如果指定 --save，就会自动保存依赖情况到Package.json中,否则只会安装包，               但不会保存package.json
            在项目根目录下直接执行 npm install将会去寻找 package.json，并且按照其中的dependencies，
            一个一个去安装好(就像是Gemfile)
        2.npm init            
            可以通过 npm init来初始化一个项目，同时也会创建package.json

4.express介绍与安装
    是一个web开发框架
    起步hello world:
        1.安装 npm install -S express
        2.引包
        3.创建你的服务器应用程序,相当于是之前的http.createServer()
    简单示例:
var express = require('express');
var app = express();
app.get('', function(req, res){
    res.send('hello express!')
});
// 利用get方法可以创建更多的 用于监听请求的代码
app.get('/about', function(req, res){
    res.send('这是关于页!')
});
// 当收到 get请求 且路径为 /的时候，执行回调处理函数
app.listen(3000, function(){
    console.log('app is running at port 3000');
})
使用框架的优点非常多，很多细节框架已经帮我们做好，就比如之前需要我们手动设置的chartset，这里它已经自动帮我们设置好了
公开指定目录:
    app.use('/public/', express.static('./public/'))
// 公开指定目录，只要这样做了，你可以通过url /public/xx 的方式直接访问 public中的所有资源了
app.use('/public/', express.static('./public/'));
app.use('/static/', express.static('./static/'));
