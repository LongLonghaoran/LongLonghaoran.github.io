Tutorial: create a module
笔记本：笔记
1.创建module,创建一个叫greetings的module
mkdir greetings && cd greetings && go mod init example.com/greetings
vim greetings.go:
package greetings

import "fmt"

// Hello returns a greeting for the named person.
func Hello(name string) string {
    // Return a greeting that embeds the name in a message.
    message := fmt.Sprintf("Hi, %v. Welcome!", name)
    return message
}
2.创建另一个module,来调用greetings module
cd && mkdir hello && cd hello && go mod init example.com/hello
go mod init会创建go.mod文件，用于跟踪代码的依赖信息
vim hello.go:
package main

import (
    "fmt"

    "example.com/greetings"
)

func main() {
    // Get a greeting message and print it.
    message := greetings.Hello("Gladys")
    fmt.Println(message)
}
这是一个可执行应用,所以放在main包中
3.由于greetings还未发布到,所以需要使用本地的module路径来指明模块路径
go mod edit -replace example.com/greetings=../greetings
这时候go.mod应该为如下内容:
module example.com/hello

go 1.16

replace example.com/greetings => ../greetings
4.执行go mod tidy:
go mod tidy的作用是将代码中依赖的但是还没有加入跟踪的依赖同步到go.mod中跟踪
由于刚刚添加了replace指令指定了greetings的本地路径，所以go工具这次不会去线上查找
该依赖的版本号,而是直接使用本地的路径,版本号也使用一个本地生成的伪版本号替代
module example.com/hello

go 1.16

replace example.com/greetings => ../greetings

require example.com/greetings v0.0.0-00010101000000-000000000000

5.go run hello.go    然后就可以执行使用了

6.compile and install application
通过go list -f '{{.Target}}'可以检查go install的目标安装路径
在当前目录下执行go install会将应用编译并安装到目标路径
