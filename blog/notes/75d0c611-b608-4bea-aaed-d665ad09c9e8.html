1.命名空间,闭包模块化开发
    管理变量，防止污染全局，适用于模块化开发,解决多人开发时变量命名冲突的问题。
    * 很久以前是这么解决的:    
        var obj = {
            department: {
                longhr: {
                    name: 'abc'
                },
                ruyuan: {
                    name: 'cde'
                }
            }
        }
        当需要使用时去点出来:    obj.department.longhr.name
    * 使用闭包来解决:    
        var initMyWork = (function(){
            var a = 1;
            function work1(){};
            function work2(){};
            return function(){
                work1();
                work2();
            }
        })();
        定义一个立即执行函数，在闭包内编写自己的业务，定义自己需要的各种属性，
        在最后再返回初始化函数即可,在闭包内写任何变量和定义任何方法都不会和别人
        命名冲突，因为返回函数拥有自己独立的作用域了，而且将闭包内的整个作用域都带出来了。
2.模拟jquery的链式的调用:
    jquery中常见方法调用形式:
        $("div").width('100px').height('100px');
    在js中可以这么写:
        var obj = {
            m1: function(){console.log(1);return this;},
            m2: function(){console.log(2);return this;}
        }
        obj.drink().smoke();

3.调用对象属性可以采用 obj['name'] 的形式:
    var obj = {
        name1: 'name1',
        name2: 'name2'
    }
    for(i=1; i<3;i++){
        console.log(obj['name'+i])
    }
    这种形式调用起来很灵活

4.对象的枚举
    for(in)循环，for in循环时专门用来遍历对象的属性的循环
    var obj = {
        name: 'longhr',
        age: 18
    }
    for(var i in obj){
        console.log(i + ' ' + obj[i])
    }
    一般都会和obj.hasOwnProperty(i)配合使用，该方法用来判断属性是否是自身直接定义的属性，如果是原型链上父级的那些属性就不算自身的属性。
    
    in 操作符,判断对象是否可以调用这个属性，比如: lastName in obj

7.this
    在预编译过程中，this指代window对象，包括在函数中也是如此
        function test(){
            this.a = 1;
            this.b = function(){console.log('这是函数内部定义的函数')}
        }
        test();
        当运行完test之后，直接在控制台调用 a b()或者调用window.a window.b()，
        会发现a b已经绑定在window对象上了。
        原理:
            * 函数中的this指代window对象。
            function test(){
                a = 1;
                b = function(){console.log('这是函数内部定义的函数');}
            }
            * 但是当以new 方式使用时例外，new的时候，函数体中的this表示当前对象，因为它会先隐式的创建一个 this ={},最后再 return this，已经更改了this的指向
            *  定义构造函数时的属性时，this.是必写的，否则属性会定义到window对象上去，因为任何未经声明的变量都是从window对象上调用
