2.定制bootstrap
    bootstrap是用Less来编写的，而使用less最好是将代码分块书写
    variables.less 存储变量定义
    mixins.less 用于引入所有用到的函数

    核心文件: bootstrap.less 在这个文件里引入了其余所有的文件
        如果我们需要定制bootstrap，比如不需要 dropdown.less那么，我们就去bootstrap.less中删除它的引入,           然后重新编译，当然有些组件之间会有依赖效果，所以去除的时候需要配合文档多加注意

        1.去除一部分组件或者样式
            比如我们不需要table的样式,那么在bootstrap.less中删除 @import 'table.less';
            然后使用lessc重新改文件生成bootstrap.css
        2.更改栅格系统的列数
            我们希望栅格系统增加到24列
            那么，来到variables.less中修改@grid为24，然后重新编译即可

4.一种新的适配方案 使用less + rem来完成页面的适配

6.使用rem时计算当前基准值:
    因为: 预设基准值 / 当前基准值 = 设计稿宽度 / 当前设备宽度
    所以: 当前基准值 = 预设基准值 * 当前设备宽度 / 设计稿宽度
    由此可知，为了得到当前基准值（也就是html元素的font-size），我们需要知道当前设备宽度 设计稿宽度 预设基准值，而预设基准值和设计稿宽度是我们一开始就知道的，至于当前设备宽度，可以通过js或者直接
媒体查询获取

7.less数组变量以及遍历数组
    less没有循环语法，像sass更高级的语言有
    但是可以使用它的递归实现近似循环的效果，函数定义时还可以附加一个"when (条件)"当条件成立时才执行
   定义函数: 
/*适配函数*/
.adapterMixin(@index) when (@index > 0){
  @media (min-width: extract(@adapterDeviceList, @index) ) {
    html {
//    extract方法用于获取数组变量中指定下标的值
      font-size: @baseFontSize/@psdWidth * extract(@adapterDeviceList, @index);
    }
  }
  //给函数的执行附加条件
  .adapterMixin(@index - 1);
}
调用函数:
/*适配器*/
.adapterMixin(@len);
