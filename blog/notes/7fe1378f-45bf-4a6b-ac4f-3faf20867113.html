puppet6主从机快速实践
笔记本：Puppet
0.配置域名解析，需要保证客户端和服务端之间仅通过名称就能够互相访问
    服务端和客户端配置相同的域名解析，将服务端和客户端的名称都配置好，puppet客户端在与服务端通讯的时候，默认会使用puppet这个主机名，所以我们这里直接配置puppet对应的ip地址即可
    vi /etc/hosts
        主服务端的ip地址 puppet

1.主服务机器
  1.  sudo rpm -Uvh https://yum.puppet.com/puppet6-release-el-7.noarch.rpm
  2. yum install puppetserver
  3. systemctl start puppetserver
  4. systemctl status puppetserver 查看puppetserver启动状态
2.agent客户端
  1.  sudo rpm -Uvh https://yum.puppet.com/puppet6-release-el-7.noarch.rpm
  2. yum install puppet-agent
  3. sudo /opt/puppetlabs/bin/puppet resource service puppet ensure=running enable=true
  4. puppet config set server 你设置的主服务的主机名+域名(默认是puppet)
        

3.签发ssl证书
    在客户端执行 puppet ssl bootstrap
    在服务端执行 puppet ca sign —all 对所有发送到服务端的csr请求进行签名
    再次在客户端执行 puppet ssl bootstrap

4.测试建立文件
    在主服务端编辑manifest文件:
        vi /etc/puppetlabs/code/environments/production/manifests/site.pp
        输入以下内容，该配置表示在客户端上建立一个文本文件:
            # 在从机上创建文件
file { '/tmp/puppet-test.txt':
  ensure => file,
  content => 'This is a test file',    # 文件内容
}

    在客户端执行: puppet apply -t
    检查客户端上该文件是否创建成功
