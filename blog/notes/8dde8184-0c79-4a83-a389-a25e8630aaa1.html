    2.自动打包
        使用webpack-dev-server
        1.由于webpack-dev-server依赖本地的webpack,所以先本地安装webpack
            npm i webpack -D
            npm i webpack-dev-server -D
        
        2.配置webpack.config.js
        
// 由于webpack是基于nodeJs构建的，所以可以使用nodeJs的核心类库
var path = require('path');
// 向外暴露一个配置对象，将来webpack会加载它
// 如果我们执行webpack或者webpack-dev-server时，没有提供参数
// 那么就会检查根目录中是否有该配置文件，如果有就会读取
module.exports = {
    entry: path.join(__dirname, 'src', 'main.js'),
    output: {
        path: path.join(__dirname, 'dist'),
        filename: 'bundle.js'
    }
}

        3.直接执行webpack-dev-server命令
            webpack-dev-server
            使用webpack4.4.1版本，可能会提示还需要安装webpack-cli，输入yes安装即可
        4.启动后它会自动编译入口文件main.js到bundle.js，并将bundle.js输出到内存中，
            在浏览器访问域名根路径下的bundle.js可以直接访问到输出的文件
        5.安装html-webpack-plugin
            // 在内存中根据指定的模板页面，生成一份内存中的首页,同时自动将打包好的bundle.js注入到页面底部
            npm i html-webpack-plugin -D
        6.配置html-webpack-plugin
    plugins: [
        new htmlWebpackPlugin({
            template: path.join(__dirname, 'src', 'index.html'),
            filename: 'index.html'
        })
    ]
        7.使用url-loader处理图片文件(.jpg .png .gif后缀的文件)
            默认情况下webpack无法处理css文件中的url地址,不管是图片还是字体
            它依赖于file-loader
        1.安装  
        npm i url-loader file-loader -D
                2.配置,module中增加
            {
                test: /\.(jpg|png|gif)$/,
                // 在使用的loader后的?后面可以直接写参数
                use: ['url-loader?limit=7631&name=[name].[ext]']
            }
            3.默认情况下，会将图片转为base64格式，作为值用来替换原先的 url(xxx)值
            4.可以在loader名称后面加上 ?limit=7631 ，这样将会传递 limit=7631作为参数给loader,表示当图片
                大于或等于7631字节，就不解析为base64,而是处理为url路径
            5.loader后的参数 name=[name] 表示不会更改打包后的文件名
            6. .[ext]表示不会更改打包后的文件后缀名
                注意！这样会有问题：当文件名相同的两个文件（他们所处的路径不一样），它们最后打包出来的结果
                会是一样的，最后会导致所有同名的文件都变成最后一次打包的那张图片
            7.将打包后的文件名改为 hash + 原文件名,     将loader改为
                 ['url-loader?limit=7630&name=[hash:8]-[name].[ext]']
        8.使用url-loader引用字体图标, 使用 bootrap中的字体(处理.ttf .svg .eot .woff .woff2)
                配置：
            {
                test: /\.(ttf|svg|eot|woff|woff2)$/,
                // 在使用的loader后的?后面可以直接写参数
                use: ['url-loader']
            }

2.webpack和npm中的几个问题的说明
    1.json里面不能写注释，注意不要在package.json文件使用 // 等注释符号去注释某一行
    2.执行 npm install会按照package.json中指定的包去安装
    3.如果项目已经npm install了，但是运行项目有各种报错，那么可能是因为之前npm install到一半被终止了
        导致包结构不完整，这个时候可以将node_modules目录删除，然后重新npm install试试
        

3.使用babel，用于将 ES6代码转换为ES3代码,webpack中只能处理一部分es6语法，一些更高级的es6 es7
        语法，webpack处理不了，这时候需要借助于第三方的loader,将高级语法转为低级语法，再将结果交
        给webpack去打包
        1.安装
            babel-core babel-loader babel-plugin-transform-runtime 这3个包
            babel-preset-env babel-preset-stage-0
            
        2.配置webpack.config.js
            在module的rules中添加一个新的匹配规则
            配置规则的时候，必须通过exclude选项排除node_modules目录下的js,因为安装的第三方包，已经是编译
            好的文件了，无需再次打包编译，而且如果去编译的话，会非常耗时间，因为node_modules中的包通常
            都非常多
            {
                test: /\.js/,
                use: 'babel-loader', exclude: /node_modules/
            }
        3.在项目根目录新建 .babelrc 配置文件,该配置文件存放json格式的配置(所以也必须符合json的要求:不能有注释，字符串要用双引号)
{
    "presets": ["env", "stage-0"],
    "plugins": ["transform-runtime"]
}

        var login = {
            template: '<h1>这是登录组件</h1>'
        }
        var app = new Vue({
            el: "#app",

6.在webpack构建的项目中使用vue
    1.安装vue
        npm i vue安装为运行依赖
    2.导入完整版的Vue
        1.导包
// 导包,这样导入的是一个运行时的包(可查看vue包中的package.json的main属性，它指向的是一个runtime.js),
//并非最全的包，不含编译功能
import Vue from 'vue';
var vm = new Vue({
    el: "#app",
    data: {
        msg: 123
    }
});
        2.解决导入的包为运行时，而不是完整包的问题
            在package.json配置对象中加入:
    // 更改vue导入时使用完整版
    resolve: {
        alias: {
          'vue$': 'vue/dist/vue.esm.js' // 用 webpack 1 时需用 'vue/dist/vue.common.js'
        }
    }
        关于 esm.js 还是 common.js
        这是取决于我们在main.js中导入vue的方式，如果采用的ES6 import导入的话，那么就使用esm.js，如果
        采用nodeJs的require规则，也就是commonJs,那么就导入common.js
    3.  导入运行时版的Vue 以及使用
        因为并不推荐导入完整版的Vue，这样页面还需要渲染比较耗时间，而且完整版的包本身体积也要更大
        一般放到正式环境的代码都是编译好的，所以就像之前处理 less scss es6一样，使用loader去加载vue代码
        导入运行时版:
            import Vue from  'vue'
    4.建立 xxx.vue文件，写入组件代码
        将之前的页面的代码放入vue文件中
        在main.js中:
            import login from 'login.vue'
    5.配置loader编译vue
        1.安装
  • npm i vue-loader vue-template-compiler -D
  • vue-loader内部依赖vue-template-compile
  • vue-template-compiler 要和vue的版本一致
            注意:
                为了和教学视频一致，在安装这些包的时候应该在包名后面加上 @版本号，来安装同版本的包
        2.配置文件中新增loader配置项
            {
                test: /\.vue/,

            }
