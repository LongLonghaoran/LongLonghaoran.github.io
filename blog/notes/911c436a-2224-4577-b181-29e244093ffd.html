笔记本：前端学习
1.对DOM的增删改查、遍历节点树
    document代表的是整个文档,其下有一些方法用于选择元素。
    document.getElementsByTagName('xx')这是最通用、用的最多的方法，每个浏览器都可以用
    document.getElementsByName 以前只有input等表单组件才可用
    document.getElementById 一般建议尽量不用

    除了ById以外，其余的elements选择后都需要通过[i]下标的形式来获取元素的引用。
    例子:
        div = document.getElementsByTagName('div');

    querySelector 可以像css选择器一样选择元素
    querySelectorAll 可以像css选择器一样选择元素,只不过选出的是个集合
    但是，一般不用这两个方法。因为它选出的元素不是实时的，只是选择出来的元素的副本。
    <body>
        <div>a</div>
        <div>b</div>
        <div>c</div>
        <script type="text/javascript">
            //使用querySelectorAll只是选出副本
            var divs = document.querySelectorAll('div');
            //这种才是实时的
            var divs = document.getElementsByTagName('div');
            
            console.log("div集合:" + divs);
            var newDiv = document.createElement("div");
            newDiv.innerHTML = '123';
            document.body.appendChild(newDiv);
            console.log("div集合:" + divs);

        </script>

    </body>

    遍历节点树:
        * parentNode,获取父级元素,一直获取到最上级的话,最上级为#document文档对象
        * childNodes子节点们(包括各种类型的节点)
        例子:
            <div>
                <!-- this is comment -->
                <strong></strong>
                <span></span>
            </div>
            <script>
                var div = document.getElementsByTagName('div')[0];
                console.log(div.childNodes.length);
            </script>
        答案是7,元素本身算一个节点，两个元素之间不管是否有文本都算一个文本节点。

        什么叫节点？节点的类型有哪些？
            节点类型:
                元素节点
                属性节点
                文本节点
                注释节点
                document
        * firstChild 第一个子节点
        * lastChild 最后一个子节点
        * nextSibling/previousSibling 后/前一个兄弟节点,注意它所谓的节点还是和刚才一样，包括注释和文本
        节点在内，所以返回后一个节点多数情况下都是一个#text,可以使用previousElementSibling方法

        上面几种方法都是获取“节点”，因此提供了与之对应获取“元素节点”的方法:
            parentElement,最多可以获取<html>节点，#document不算元素节点
            children 所有元素子节点
            firstElementChild
            lastElementChild
