4.响应式布局
    由于不同终端的尺寸不一致，我们通常需要开发多套页面用以给不同终端展示（之前的流失布局虽然能在不同尺寸改变html元素宽度，但是它只是适合一套终端来展示，比如开发的移动端页面（比如京东），就算能够在pc显示，但是它并不好看，如果需要在Pc上显示，我们还是会需要单独来写一个专门展示pc端的页面）。所以就有了响应式开发的概念，让我们可以开发一套页面以此在不同终端展示出不同的效果。

5.原理
    媒体查询，通过查询screen的宽度来指定某个宽度区间的网页布局
        超小屏幕(移动设备) 768px以下, 一般来说ipad的宽度就是768px，所以也就以此划分了
        小屏设备 768-992px
        中等屏幕 992-1200px
        宽屏设备 1200px以上

6.响应式容器标准:
    container以这个容器为例，假设该容器是页面的一个版心的大盒子
    1.在超小屏下，当前容器的宽度应该是100% 蓝色
    2.在小屏设备的时候，当前容器宽度的750px,也就是比小屏的最小宽度还要小一点 绿色
    3.在中屏设备的时候 当前容器的宽度970px 红色
    4.在大屏设备的时候 当前容器宽度1170px 黄色

7.使用媒体查询

@media screen and (max-width: 768px){
.container {
width: 100%;
background-color: blue;
}
}
@media screen and (max-width: 992px) and (min-width: 768px){
.container {
width: 750px;
background-color: green;
}
}
@media screen and (max-width: 1200px) and (min-width: 992px){
.container {
width: 950px;
background-color: red;
}
}
@media screen and (min-width: 1200px){
.container {
width: 1170px;
background-color: yellow;
}
}
    在此基础上，仍然遵循了css样式覆盖的原则，即 如果同时有多个可以生效的样式，优先使用最后声明的样式,也就是说上面的css也可以像下面这样写

@media screen and (min-width: 1200px){
.container {
width: 1170px;
background-color: yellow;
}
}
@media screen and (max-width: 1200px){
.container {
width: 950px;
background-color: red;
}
}
@media screen and (max-width: 992px){
.container {
width: 750px;
background-color: green;
}
}

@media screen and (max-width: 768px){
.container {
width: 100%;
background-color: blue;
}
}

9.关于Normalize.css重置样式库
    之前也写过重置样式库(reset.css)，bootstrap也使用了一个，为了增强浏览器表现一致性，使得在各个浏览器中表现一致
    比如:
        去除所有a标签中的文字的下划线、ul ol列表的列表项前面都去除标记、input文本框去除outline轮廓
    但是，Normalize，不会重置那些已经在每个浏览器中表现一致的样式，比如 ul ol，在各个浏览器中list-style的样式都是一致的

10.布局容器
    .container
     .container 类用于固定宽度并支持响应式布局的容器。
    它所实现的功能也就是我们上面所实现的一长串带有媒体查询的响应式css
    .container-fluid
    .container-fluid 类用于 100% 宽度，占据全部视口（viewport）的容器。
    顾名思义，它是流式布局的百分比宽度
