1.http协议（为ajax做准备）:
    1、作用
        规范了数据是如何打包的 以及 数据时如何传递的
    2、Message 消息 / 报文
        Message 指的是在HTTP客户端与服务器间传递的数据块
        分类：
            1、Request Message : 客户端向服务器发送的请求消息
            2、Response Message : 服务器端根据客户端的请求消息，返回给客户端的响应消息
    3、请求消息(Request Message)
        1、请求起始行(也就是Headers部分显示的第一行内容)
            1、请求方法
                1、GET 常用
                    表示客户端向获取服务器上的资源(img/js/..)
                    特点：
                        1、无请求主体
                        2、依靠地址栏传递数据给服务器
                2、POST 常用
                    表示客户端想传递数据给服务器
                    特点：
                        1、有请求主体
                3、PUT
                    表示客户端想把文件放到服务器上(禁用)
                    特点：
                        1、有请求主体
                4、DELETE
                    表示客户端想删除服务器上指定的文件(禁用)
                5、HEAD
                    表示只想获取指定的响应头
                6、CONNECT
                    测试连接
                7、TRACE
                    追踪请示路径
                8、OPTIONS
                    选项，保留以后使用... ...
            2.请求头
                记住理解一些主要的就可以了
                    1.Host: 告诉服务器请求哪一个虚拟主机
                    2.Connection:Keep-alive 告诉服务器做持久链接
                    3.User-Agent: 用户代理的信息，客户端的浏览器信息
                    4.Cache-Control:max-age=0告诉服务器缓存的时间，max-age=0即表示不缓存
                    5.Accept-Language:zh-CN服务器能够接收的自然语言
                    6.Accept-Encoding: gzip告诉服务器客户端支持接收的压缩类型
                    7.Referer:url 告诉服务器请求来自于哪个页面
            3.请求主体
                form中带的表单数据体即为请求主体 form data
            4.查询字符串
                query string: 不算做请求主体，是属于url的部分
    4.响应的消息(Response Message):
        1.响应起始行(也就是Headers部分显示的第一行内容)
            1.协议版本号
                http/1.1
            2.响应状态码
                1.1xx
                    提示消息
                2.2xx
                    200: 服务器端成功响应所有信息 ok
                3.3xx 需要客户端进行重定向
                    301: 永久性重定向
                    302: 临时重定向
                    303: Not Modified
                4.4xx
                    客户端请求错误
                    404: Not Found,请求资源不存在
                    403: 没有权限,没有访问权限
                    405: Method Not Allowed: 请求方法不被允许
                5.5xx
                    服务器内部错误
                    500:
                    501: 没有实现
            3.原因短句
        2.响应消息头
            1.Content-Length: 9 响应的数据长度
            2.Content-Type: text/html;charset=utf-8
                响应的主体类型，告诉浏览器，响应的数据是什么格式什么类型的,text/html就是表示响应的数据是html页面。具体还有其他的一些类型。
                    text/plain: 纯文本,也就是说响应中即便有html标记也会按普通文本去处理。
                    text/html: 文本与网页
                    text/css: 样式，告诉浏览器按css的方式解析
                    application/javascript js代码，告诉浏览器按js的方式运行数据
                    application/xml: 按xml的方式解析
                    application/json: 按json的方式解析
        3.响应主体
            由服务器端响应回来的数据组成,即浏览器的preview和 response两个tab的内容
2.缓存
    1.工作原理
        客户端可以自动保存已访问过的文档的副本，这些副本就是缓存
        当客户端再向同一个uyrl发送请求时，那么就直接从缓存中将文件取出来，而不用再重新发送请求。

    2.优点
        1.减少了冗余的数据传输，节省流量
        2.缓解服务器带宽瓶颈的问题，可以节省更多的带宽
        3.降低了对服务器的资源消耗和运行要求
        4.降低了由于远距离造成的加载延迟，提升了用户体验
    3.与缓存相关的消息头
        1.Cache-Control
            作用：从服务器将文档传来之时起，认为新鲜的秒数(过了多少秒之后就过期)
            取值：秒数 60 3600 ...
            如果取值为0/no-cache，表示不进行缓存，每次都从服务器进行获取
        2.Expires，相比起Cache-Control,它明确指定缓存的过期时间，取值时格林尼治时间(GMT),当Expires设置为0，则表示不需要缓存。
    4.如何在网页中设置消息头
        <head>
            <meta charset="utf-8">
            <meta http-equiv="Content-Type" content="image/png">
        </head>

        http-equiv指定消息头名称，content指定消息头对应的内容

5.HTML元素的事件
    事件：在某些特殊的情况下，可以被触发的一个操作

    1.文本框 和 密码框
