笔记本：前端学习
1.注意点：
    1.行内元素内部尽量只包含行内元素和文字
    2.只有文字才能组成段位，因此p里面不能放块级元素，同理还有h1 h2 h3 ..dt
    3.链接里不能再放链接
2.行高的测量
    测量:每行都由4条线组成，顶线 中线 基线 底线，行高是2行的基线之间的距离，文字垂直对齐方式，设置 line-height = height

3.css三大特性
    1.层叠性
        多个选择器同时设置相同的属性时，以最后的样式为准
    2.继承性
        子标签会继承父标签的“某些”样式，比如字体颜色color,字体大小 字体
    3.优先级(难点)
        对于同一个元素而言，如果有多个规则定义在一个元素上,会按照优先级依次取样式
        基本概念:
            */继承的选择器 0000< 标签选择器 0001< 类选择器 0010< id选择器 0100< 行内 1000< !important ∞
        权重是可以叠加的:

    * 当2个样式权重相同时，按就近原则取较近的样式
    * 当选择器是个复合的选择器时，权重叠加计算,
        比如 div p,这样的话，权重为 0002
        .feng p,这样的权重等于 0010 + 0001 = 0011
    * 权重的数位之间没有进制，比如 0001 + 0001 = 0002，而不会进位0010,所以不会存在说10个div能赶上一个类选择器的情况
    *继承过来权重是最低的，即便父标签的属性加上了!important

4.css背景
    background-repeat: repeat no-repeat repeat-x repeat-y

    background-position: 设置或检索对象的背景图像位置。必须先指定background-image属性。默认值为：(0% 0%)。
        如果只指定了一个值，该值将用于横坐标。纵坐标将默认为50%。第二个值将用于纵坐标。
        该属性定位不受对象的补丁属性(padding)设置影响。

    background-attachment:
        设置或检索背景图像是随对象内容滚动还是固定的。
        对应的脚本特性为backgroundAttachment。请参阅我编写的其他书目。
        语法：
        background-attachment : scroll | fixed  
        参数：     
        scroll : 　背景图像是随对象内容滚动
        fixed : 　背景图像固定  

    background-image:
        设置或检索对象的背景图像。
        参数：
        none : 　无背景图
        url : 　使用绝对或相对地址指定背景图像
        
        说明：
        
    background:
        如使用该复合属性定义其单个参数，则其他参数的默认值将无条件覆盖各自对应的单个属性设置。
        默认值为：transparent none repeat scroll 0% 0%。

5.css3 背景透明
    background的第一个参数就表示透明度，用rgba()表示，rgba(x,x,x,x)前3位表示颜色，第4个是透明度，取值范围0~1

6.css盒子
    border边框
        border : border-width || border-style || border-color
    padding内边距
    margin外边距

7.表格细线边框
    当表格中每个单元格都设置了1px的边框时，那么实际展示出来的效果将会是2px的
    边框，这时可以通过属性border-collapse: collapse来处理
    table {
        border-collapse: collapse;
    }

8.外边距padding:
    padding: 上 右 下 左
    由于添加padding导致盒子尺寸发生变化的解决方案,
    一般我们会给盒子设置width height,这个时候我们会期望
    盒子最终就是这么大的，之后为了显示好看，我们会继续
    添加padding属性，但是加了padding之后，盒子尺寸会被撑开，这个时候就需要重新设置一下width height了。

9.外边距margin:
    margin:  上 右 下 左

10.* {padding: 0;margin: 0}
    body元素默认会有margin: 8px,导致内容无法顶边显示，一般可以在样式最开头添加*{margin: 0px;padding: 0px}将所有框的内外边距去除。

11.盒子的水平居中对齐，即盒子本身在其父标签中居中对齐。
    margin: 0 auto;注意要先指定盒子宽度

12.垂直外边距合并:
    当两个垂直方向的元素在一起时（比如2个div)，处于上方的元素有下方的外边距，处于下方的元素有上方的外边距，那么最终2个盒子的距离并不会等于外边距之和，而是2者之中较大者。解决方案，避免同时设置。

13.外边距塌陷:
    当存在有嵌套关系的2个盒子，希望给子级盒子一个上外边距，margin-top: 50px,这样子是没用的
    解决方案:
        1.给父元素设置一个上边的边框
        2.给父元素设置上内边距
        3.给父元素设置 overflow: hidden;

14.padding不会撑开盒子的情况:
    一个盒子如果没有指定宽度，则会沾满父亲的宽度，而且padding不会对其宽度造成影响。

15.圆角边框(css3)
    border-radius: 值
    值越大圆度越高,对于一个方形矩形而言，当圆度为
    width一半时，就表示一个“正圆”,值也可以直接写成50%,
    也表示正圆。
    border-radius也可以指定4个值，分别指代：
    左上 右上 右下 左下

16.盒子阴影 box-shadow

17.浮动 float
    网页布局的核心，就是盒子的摆放。
    css定位有3种机制： 普通流/标准流， 浮动流， 定位

    浮动:
        float: left/right/none
        只有left right两种，没有top bottom之说

    用浮动的原因：一方面可以让div在一行显示，另外使用
    浮动的话，div之间不会有边距，如果使用inline-block 的话，就算div显示在同一行，但是div之间会有边距

    特性:
        * 浮动会脱离标准流，是不占位置的。
        * 元素的浮动是在其父级元素内部进行浮动的,浮动的位置也是相对其父级元素的,而且浮动后所在位置的
            界限也仅限于父级元素的内容界限内，即不包含border padding在内的content的范围。
        *浮动影响的流仅仅只是它身后的标准流。
        *浮动会让元素默认转换为行内块元素,也就是说,如果不指定其宽度的话，那么它的宽度会变得和内容一样宽
        * 浮动的目的是让块级元素在一行内显示
