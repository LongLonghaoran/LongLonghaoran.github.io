笔记本：前端学习
1.ajax
    首先除了ajax还有其他方式可以发送请求，并且获取数据
        var img = new Image();
        img.src = '1.jpg';
    除此之外，也就是
        window.reload()
        location.href = 'http://www.baidu.com'
    但是很明显，这些方式都没有获取请求的返回结果,只有ajax可以做到这点

    ajax介绍:
        1.其实就是一套api,可以通过js调用，从而实现通过代码控制请求与响应
2.ajax使用
    通过使用XMLHttpRequest来实现ajax, XML是由于历史原因，最初数据传递是用xml来表示的，
    Http即Http协议,Request就是请求,所以起名叫做XMLHttpRequest

    XMLHttpRequest在Js中也是一个构造函数，可以通过它来新建ajax对象。
    使用ajax的一个基本的例子如下:
    1.新建一个代理对象
        var xhr = new XMLHttpRequest();
    2.设置网址
        xhr.open('get', "time.php", true);
        第三个参数表示是否为异步请求,设为true就是异步请求，使用ajax几乎全是异步的，因为如果是同步的话，
            那么页面有可能因为接口的卡顿而导致整个页面卡死
        需要注意，这个网址也可以写成其他网址的，而且请求的端口也可以调整
            xhr.open('get', "http://www.sss.com:8080/time.php")

    3.发送请求，等待响应,执行到这一步的时候，才会发送请求
        xhr.send();
    4.接收响应,并不能直接接收响应结果，这很好理解，因为，我们发送的请求，不可能立马收到响应，根据网速、响应数据大小，返回响应的时间长短不一定
       我们可以通过事件绑定，通过回调函数来接收响应
       xhr对象有一个事件onreadystatechange，用来绑定回调函数
       它的作用是，一旦异步请求发送状态变化就会调用一次回调函数,函数中的this
       还是和之前一样，代表着事件的调用着，异步请求对象xhr
        xhr.onreadystatechange = function(){
            通过readyState属性即可判断请求处于哪个状态
            其实总共有5个状态 0 1 2 3 4
            0 --- new XHR 创建
            1 --- open 打开请求
            2 --- send 发送请求
            3 --- 接收请求
            4 ---
        }
        1.readyState
            如果我们按照上面的顺序书写代码，能够捕获 2 3 4三个状态，
            注意，1也是能够捕获到的，这是因为事件来不及捕获！如果
            我们将onreadyState放在open前面书写的话，那么它将可以捕获到1的事件
            0也是捕获不到的，因为readystatechange事件只有当状态发生变化才会触发，
            而0是个初始状态

            当readyState == 4时就代表请求已经完全结束
        2.responseText 用于获取响应的响应体body
        解析各个readyState:
        2. readyState == 2
            当readyState为2时，说明可以接收到响应头了,通过
            xhr.getAllRequestHeaders()方法可以获取响应的头信息，以字符串表示
            还有另外一个方法，可以用来获取指定的键的值
                getResponseHeader(key);
            它接收可以一个参数，表示获取指定键的值，比如：
                xhr.getResponseHeader('server');
            顺带一提，通过server获取的值，表示服务器的一些信息，包括服务器名称 还有对应的后端语言名称，
            很多检测网站插件的软件，实质上就是通过检测它的header来检查的
        3.当readyState == 3
            这个时候表示正在下载，这个时候其实也可以拿到responseText了，
            但是由于它还正在下载，所以有可能拿到的响应体不完整
        4.当 readyState == 4
            表示整个响应报文已经完整下载下来了,这个时候去拿
                header responseText都是可以的
        5.设置请求头
            xhr.setRequestHeader("content-type", "application/json")

        6.事件注册最好是在 send请求发送之前注册，这样更加可靠

        7.send方法可以传递 请求参数,设置请求体
            1.传递 request payload
                send("a=1&b=2")
                这个时候传递的request-payload 将会是 a=1&b=2
                当form设置了 enctype="multipart/form-data" 的时候
            2.传递form-data
                需要设置请求头
                xhr.setRequestHeader("content-type", "application/x-www-form-urlencoded")
        8.status 获取状态码
          statusText 获取状态的描述

        9.一种比较规范的判断 readyState状态的方式:
            xhr.onreadystatechange = function() { // Call a function when the state changes.
                if (this.readyState === XMLHttpRequest.DONE && this.status === 200) {
                    // Request finished. Do processing here.
                }
            }
            XMLHttpRequest.DONE相当于是个常量 等价于 4， status是状态码

        10.JSON.parse 用于将json格式的字符串转为 对象形式

        11.如果ajax指定为同步请求的(也就是open方法第三个参数为false)，那么就可以不用绑定回调事件了,可以直接通过
            xhr.responseText获取响应体
            
    5.onload事件就等于 readyState == 4的事件,但是onload是h5新增的事件有一定的兼容性的问题

    6.详解 ajax请求的参数
        1.get 请求的参数,参数放在 url后面 ?xx=xxx即可
        2.post请求
            1.发送页面中的表单的数据,做到页面不刷新即可提交表单数据
                1.注意设置请求头  xhr.setRequestHeader("content-type", "application/x-www-urlencoded");


    7.额外的知识:
        1.
            ajax请求过后如果得到的响应体是个xml的话，可以通过xhr.responseXML来访问/遍历xml文档
            并且，通过documentElement可以进一步拿到xml的dom对象，然后就完全可以当做dom对象来处理:
                之后可以通过 getElementByXXX来获取dom元素
                xhr.responseXML.documentElement.getElementByTagName("name");
        2.模板引擎 arttemplate
            通过ajax获取数据后，如果需要将数据渲染到页面中，如果每次都是手动创建html元素
            来创建的，是非常麻烦的
            这里介绍个arttemplate 插件,模板引擎
            ，作为今天的任务，自学一下
