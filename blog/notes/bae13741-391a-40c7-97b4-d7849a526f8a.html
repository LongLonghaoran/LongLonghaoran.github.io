4.url模块
    
[图片/附件]

将url作为参数传递给parse方法，它可以返回一些实用的方法来解析该url
    1.protocol 协议
    2.slashes 判断 // 如何处理，从第一个//开始之后的部分为路径
    3.query 查询参数，如果parse方法第二个参数与为true,那么该属性将是一个 由 查询参数组成的 js 对象,如果不是true,那么这里将是一个 查询字符串,比如 'a=1&b=2'

即便，parse方法的参数，不带前面的host 主机名 http:// 协议，仅仅只是从 path 路径开始(比如 '/a/b?c=1'
它也可以正确解析，就是说它总是解析url中正确的那一部分

5.重定向
    1.状态码设置为302 通过 statusCode
        res.statusCode = 302
    2.响应头设置 Location告诉客户端往哪儿重定向 setHeader('Location', 'http://需要重定向的地址')
        res.setHeader('Location', '/路径') 从根路径开始写即可，无需包含协议和主机名，当然写了也是可以的
    
var parseObj = url.parse(req.url, true)
var comment = parseObj.query
comment.dateTime = '2017-11-2 17:11:22'
comments.unshift(comment)
res.statusCode = 302
res.setHeader('Location', '/')
res.end()
