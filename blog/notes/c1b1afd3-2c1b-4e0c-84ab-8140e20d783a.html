Prometheus - exporter安装说明

概述

Prometheus通过各种exporter采集metrics信息，然后通过exporter开放出的web接口，拉取到server端进行存储分析。通常exporter会在特定端口（不同exporter默认端口不同）开放出/metrics路径，供prometheus server拉取监控元数据。Prometheus生态中提供很多成熟开源的exporter，可以直接根据需求使用（exporter列表），不同的exporter，都有对应grafana模板用于grafana面板监控（grafana dashboards），也可自定义面板进行监控。
系统中常用exporter如下：

node-exporter

前置准备工作

无

下载链接

https://github.com/prometheus/node_exporter/releases/download/v1.0.1/node_exporter-1.0.1.linux-amd64.tar.gz

安装说明

下载后压缩包node_exporter-1.0.1.linux-amd64.tar.gz中包含可执行程序，将压缩包上传到待监控服务器，并解压到/home/op/prometheus/node-exporter路径下

配置说明

详见https://github.com/prometheus/node_exporter

开机启动

通过systemd对exporter进行管理，实现随系统自动启动

配置文件编写

1sudo vi /etc/systemd/system/node-exporter.service
文件内容：
注意下面op是操作用户名称
1[Unit]
2Description=Prometheus Node Exporter
3After=network.target
4[Service]
5ExecStart=/home/op/prometheus/node-exporter/node_exporter
6User=op
7[Install]
8WantedBy=multi-user.target

服务启动

1#配置重载
2sudo systemctl daemon-reload
3#服务开启，“node-exporter”为配置文件名前缀部分
4sudo systemctl enable node-exporter
5#服务启动
6sudo systemctl start node-exporter


#通过journalctl可以查看服务启动日志(如果启动失败的话)
journalctl -u node-exporter.service

前置准备工作

需申请数据库用户：
用户名：exporter
用户权限：
1CREATE USER exporter@172.16.22.51
IDENTIFIED BY ‘cscsitme' WITH MAX_USER_CONNECTIONS 3;
2GRANT PROCESS, REPLICATION CLIENT, SELECT ON *.* TO 'exporter'@'待部署exporter客户端服务器IP';


CREATE USER exporter@10.3.2.71 IDENTIFIED BY 'cscsitme' WITH MAX_USER_CONNECTIONS 3;
GRANT PROCESS, REPLICATION CLIENT, SELECT ON *.* TO 'exporter'@'10.3.2.71';


下载链接

https://github.com/prometheus/mysqld_exporter/releases/download/v0.12.1/mysqld_exporter-0.12.1.linux-amd64.tar.gz

安装说明

下载后压缩包mysqld_exporter-0.12.1.linux-amd64.tar.gz中包含可执行程序，将压缩包上传到用于exporter部署服务器，并解压到/home/op/prometheus/mysqld-exporter路径下

配置说明

详见https://github.com/prometheus/mysqld_exporter

开机启动

通过systemd对exporter进行管理，实现随系统自动启动

配置文件编写

  1. exporter配置文件
单台服务器配置多个mysqld_exporter的时候：
  • my.cnf名字可按数据库名定义，如数据一致性服务数据库的exporter：consdb-my.cnf
1sudo vi /home/op/prometheus/mysqld-exporter/my.cnf
文件内容：
1[client]
2user=exporter
3password=密码
4port=数据库端口
5host=数据库地址

  1. systemd配置文件：
单台服务器配置多个mysqld_exporter的时候：
  • mysqld-exporter.service名字可按数据库名定义，如数据一致性服务数据库的exporter：cons-mysqld-exporter.service
1sudo vi /etc/systemd/system/mysqld-exporter.service
文件内容：
单台服务器配置多个mysqld_exporter的时候：
  • my.cnf文件需改成对应客户端配置文件路径
  • 需修改web.listen-address以指定不同端口
1[Unit]
2Description=Mysql Exporter
3After=network.target
4[Service]
5ExecStart=/home/op/prometheus/mysqld-exporter/mysqld_exporter --config.my-cnf=/home/op/prometheus/mysqld-exporter/my.cnf --web.listen-address=:9104
6User=op
7[Install]
8WantedBy=multi-user.target

服务启动

单台服务器配置多个mysqld_exporter的时候：
  • 启动服务名为service配置的文件名前缀，如数据一致性服务数据库的exporter：cons-mysqld-exporter
1#配置重载
2sudo systemctl daemon-reload
3#服务开启，“mysqld-exporter”为配置文件名前缀部分
4sudo systemctl enable mysqld-exporter
5#服务启动
6sudo systemctl start mysqld-exporter
