笔记本：前端学习
1.jquery:
    学习的理由:
        1.js写代码，需要遍历
        2.找对象麻烦,方法少还长
        3.有兼容性问题，比如 innerText textContent
        4.其实在js中 window.onload方法会被覆盖，而jquery中可以用$(document).ready(function(){})多段也不会覆盖
            虽然可以使用addEventListner来解决，添加多个相同事件绑定，但是addEventListener这个方法
            又有兼容性问题，有时需要使用attachEvent，总之全部都用js来写是件很麻烦的事情

    1.注册事件
        $(document).ready(function(){})
    2.$选择器，是个函数
        有个值得注意的地方，它的原理应该 是使用的类似document.querySelector,因为通过动态选择
        元素的例子发现，它并不会死循环
                $(document).ready(function(){
                    console.log("呵呵");
        //            用getElementsByClassName会死循环
                    lis = $(".current");
                    for(var i=0;i<lis.length;i++){
                        var ele = document.createElement("li");
                        ele.className="current";
                        lis[i].appendChild(ele);
                    }
                });
        当然并非JS在模仿jquery,相反document.querySelector是在h5中才提出的，而Jquery早在h5
        以前就有这种css选择器的思想了
    3.Jquery2.0开始就不支持ie 678了，如果需要支持，就使用Jquery1.12,一直到目前为止2019年,国内大多数
        网站像 百度 京东，仍然用的Jquery 1.xx，就是为了兼容IE678
    4.jQuery的一些重大特性:
        1.选择器
        2.Jquery有一个隐式迭代
            我们通过选择器选择的元素如果一个节点集合的话，如果我们想遍历集合给每个元素做相同操作，
            在Jquery不用再去遍历了，直接把它当做单个元素调用即可，因为Jquery会自动帮我们迭代
            $("div").show()
        3.注册事件不会被覆盖
            Jquery中注册事件使用:
                $().click(function(){})
                $().on('click', function(){})
    5.Jquery入口函数,虽然有$(document).ready,但是在写代码时，仍然建议将代码写在页面底端
        有几种写法:
        1.$(document).ready(function(){})

        2.$(function(){});

        3.如果和window.onload一起使用呢?
            window.onload = function(){}
            $(document).ready(function(){})
            $(function(){});
            上面的代码中.js的入口的函数window.onload将会最后执行,而document.ready $(function)
            则是按照代码顺序来执行
        4.事实上
            jq的入口函数会等待页面的文档加载完成执行，但是不会等待图片加载完成
            而onload会等待页面加载完成，并且等待图片加载完成才开始执行
        5.jq对象和dom对象的区别 重点
            dom对象也叫js对象，叫法不一样而已
            1.这是两个不同的对象
            2.jq对象其实就是一个Js对象的集合，就是一个伪数组，里面存放了一大堆的js对象
                通过Jq选择出的Jq对象如果调用 [0]方法，可以直接获取一个js对象
            3.jq对象无法调用js对象的方法，js对象也无法调用jq的方法
            4. jq -> dom
                jq对象如果需要调用js对象的方法，需要通过它取出js对象之后才能调用
                $("div")[0]
            5. dom -> jq
                dom对象/js对象 转换成 jq对象，用$()包起来即可
                var ele = document.querySelector("div");
                $(ele).text(123)
                就算是node list也可以转换
                $(document.getElementsByTagName("div"));
            6.$到底是什么
                1.就是个函数 $ instanceof Function //true
                2.$ 其实总的来讲，它就3个用途
                    1.入口函数 $(function(){});
                        入口函数的标准形式
                    2.$(domObj) 把dom对象转换成jquery对象
                        比如$(document),document是dom对象，表示文档,然后jquery的ready方法
                    3.$(字符串) 作为元素的选择器,选择器的规则是个类css选择器,返回值是个jq对象
                        它兼容了css1 到 css3所有的选择器
            7.Jquery设置样式
                css方法: $().css("backgroundColor"， "pink")
            8.子代选择器 后代选择器
                $("div>p") 表示div 下直接子元素为p的
                $("div p") 表示div下所有为p的后代
            9.筛选选择器（方法）
                在用字符串选择器选择之后进一步使用筛选方法来筛选元素
                1.$(选择器1).children(选择器), 选择的是所有的直接子元素
                   在选择器1的基础上再次进行选择
               2.siblings(选择器) 所有兄弟元素,也可以进行选择
               3.find(选择器), 获取所有后代，也可进行选择
               4.next()下一个兄弟 -----------> js中: nextElementSibling
               5.prev()前一个兄弟 -----------> js中: previousElementSibling

            10.Jquery中给元素注册事件实际上仍然是注册给dom对象了
                $("div").onclick = function(){
                    console.log(this);
                }
                事件回调函数中的this表示的是DOM对象,所以如果我们需要让它像Jquery对象一样来使用
                应该加上$(),
                $("div").onclick = function(){
                    console.log($(this));
                }

        6.mouseover mouseenter事件的不同
            注册mouseover的元素的子元素也会触发，并且导致父元素触发冒泡，会执行多次
            mouseenter 在鼠标进入父元素后只会触发一次
