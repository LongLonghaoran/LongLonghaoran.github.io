1.try catch
    try{

    }catch(e){

    }
    try中的代码执行出错会被捕获，一旦代码出错，那么在try中出错行之后的代码不会继续执行，且代码跳转至catch中，一旦try中不出错则catch代码不会执行。
    e表示错误对象，报错时会将name和message封装至error,分别表示错误名称和错误信息。
    error.name的六种值对应信息：
        1.EvalError: eval()的使用与定义不一致
        2.RangeError:数值越界
        3.ReferenceError:非法或不能识别的引用数值
        4.SyntaxError:语法解析错误
        5.TypeError:操作数类型错误
        6.URIError:URI处理函数使用不当

2.es5严格模式:
    以前(2017年),浏览器的js语言，分为es3基础 和es5新增的部分，但是es3和es5冲突部分会采用es3的功能,es5的严格模式就是为了解决这个问题的。
    语法:一个use strict的字符串即可开启es5.0的严格模式
    "use strict";
    使用了use strict之后的代码将采用严格模式来执行。

    一般来说，规定use strict要在所有代码的最前端。但是也可以写在局部，写在局部的话，则use strict之后的代码按照es5.0来执行。
    一般分为两种用法：
        全局严格模式
        局部函数严格模式（推荐）

    es5.0严格模式的启动:
        1.严禁用with();使用的话运行速度过慢，而且代码复杂度提升
        2.禁止使用arguments.callee 和 Obj.caller
        3.禁止使用未声明的变量
        4.局部的this，必须进行赋值，预编译过程不会默认地指代window
        5.拒绝重复属性和参数
            'use strict'
            function test(name, name){

            }
            //报错
            var obj = {
                a: 1,
                a: 2
            }
            //虽然不报错，但是es5.0是拒绝这种写法的
        6.拒绝使用eval,因为它可以改变作用域，不太安全


    * 补充知识点:
        with的使用：
            with(obj){
                obj的作用域
            }
        with的作用域，将obj的作用变为括号内的作用域链的顶级作用域,在{}内执行代码，将会优先使用obj.xx的形式进行调用，obj内找不到时，才会去上级作用域查找 。比如经常使用document.xx方法，可以这样做:
            with(document){
                在这里可以不用document.，直接使用其中的方法。
            }

        eval方法，能将字符串当代码来执行

3. DOM
    DOM定义了表示和修改文档所需的方法，DOM对象即为宿主对象，由浏览器厂商定义，用来操作html和xml功能的一类对象的集合。也有人称DOM是对html以及xml的标准编程接口。
    关于xml,是html最早的一种形式，发展历程是这样的 xml -> xhtml -> html,xml的标签名称可以自定义，可以用来表示一种数据结构，现在已经基本被json所替代。

    注意DOM不能改变css,比如 ele.style.backgroundColor这样的形式去增加，实际上还是改变的dom，通过添加行内属性来改变样式而已，并没有改变css表。

    实现按键盘可以自由移动的方块:

    <div id="moveableDiv" style="width: 100px;height: 100px;background-color:yellow;top:0;left:0;position:absolute"></div>
    <script type="text/javascript">

        var moveableDiv = document.getElementById("moveableDiv");
        document.onkeydown = function(e){
            console.log(e);
            switch(e.which){
                // 对应左上右下
                case 37:
                    moveableDiv.style.left = parseInt(moveableDiv.style.left) - 10  + 'px';
                    break;
                case 38:
                    moveableDiv.style.top = parseInt(moveableDiv.style.top) - 10  + 'px';
                    break;
                case 39:
                    moveableDiv.style.left = parseInt(moveableDiv.style.left) + 10 + 'px';
                    break;
                case 40:
                    moveableDiv.style.top = parseInt(moveableDiv.style.top) + 10 + 'px';
                    break;
            }
        }
    </script>
