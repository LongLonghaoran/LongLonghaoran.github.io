1.1. 核心组件

ray serve运行在ray cluster集群中，主要由Controller, HTTP Proxy, gRPC Proxy, Replica组成，其以ray actor的形式运行：

  • Controller：ray serve的全局控制器运行在ray cluster中的head node中，其负责HTTP Proxy/gRPC Proxy/Replica的创建、更新、删除，诸如创建或获取部署之类的Serve API调用会向控制器发出远程调用。
  • HTTP Proxy：用于接收传入请求，将其转发到Replica，并在请求完成后作出响应，每一个节点可以有一个HTTP Proxy。
  • gRPC Proxy：作用于HTTP Proxy类似，提供gRPC服务。
  • Replica：实际执行响应请求的组件，如大模型响应推理请求。

一个请求的生命周期：

  1. 某用户向ray serve发送一个查询请求，如果是http请求，则被HTTP Proxy解析
  2. HTTP Proxy根据http请求的URL path（如/model1，/model2）将请求转发到不同的模型实例（这些模型实例可能分布在不同的机器）的Request Queue
  3. 接着Replica处理执行Rquest Queue中的请求，响应这个请求


1.2. Ray Serve AutoScaler

Ray Serve AutoScaler是Controller组件的一部分，能根据请求量自动扩缩容模型实例的副本数。

[图片/附件]

deployment's ServeHandle：向模型实例发送请求的应用程序

  • deployment's ServeHandle与worker replica向autoscaler发送metrics(待处理队列中的请求排队数、查询数等)
  • autoscaler根据metrics决定是否要扩缩容worker replica

@serve.deployment(

“custom_resources”可指定特定资源，可将模型部署到特定节点。对于ray cluster集群部署来说，可以为各个节点指定"custom_resources"，如：

    1. head node：ray start --head --num-cpus=3 --num-gpus=4 --resources='{"custom_label": 1, "custom_resources": 2}'
    2. worker node：ray start --address=<head-node-address:port> --resources='{"special_hardware": 1, "another_custom_resources": 2}'

3. 关键核心技术



3.1. 异构AI推理任务调度（推理一体机集群）

解决推理一体机（每个一体机上部署不同的模型）集群AI推理任务的调度问题，以及多个模型组合问题。

  1. ray cluster集群可将多个推理一体机进行统一纳管，将不同一体机纳入ray cluster时，可根据ray start --resouces='{"custom_resouces": 2}'为该推理机指定特定资源
  2. 参考ray 部署多个推理服务实例，根据ray_actor_options={"resouces": {"custom_resources": 2}}参数，可以将不同大模型部署调度到不同一体机，通过不同http URL path将请求流量转发到不同推理一体机


3.2. 动态扩缩容

解决用户流量激增问题，Ray Serve Autoscaler支持模型实例副本的自动扩缩容。

  1. 当请求流量增加时，ray serve autoscaler准备扩容模型实例的副本，判断当前集群是否有足够资源（cpu，gpu）来扩容副本，若没有，则扩容节点（对于kuberay来说则是加pod，对于一体机来说得增加一体机），将模型实例副本调度到这些节点上
  2. 当请求流量减少时，ray serve autoscaler缩减节点，同时减少模型实例副本，释放资源


3.3. 负载均衡

解决用户流量激增问题，提升用户体验。

  1. 每个ray cluster节点都有一个HTTP Proxy，可以将当前节点的流量负载均衡到当前节点的不同副本，若请求的大模型不在当前节点，HTTP Proxy会将请求转发到其他节点进行处理
  2. 对于将外部流量负载均衡到ray cluster，需依赖nginx等进行处理


3.4. 自动容错

解决节点故障问题，增强服务高可用性。

  1. 通过模型实例副本来实现故障容错

  • 本地windows机器：
    • serve run config.yaml -a ray://10.44.11.11:10001（默认client端口是10001，类似于ray.init(address="ray://10.44.11.11:10001")），serve run命令是会一直在终端输出日志，方便调试，或者
    • serve deploy config.yaml -a http://10.44.11.11:8265（默认的dashboard端口8265）
  • node11机器：
    • serve run config.yaml 或者
    • serve deploy config.yaml
