ruby中如何创建死锁的情况
笔记本：Ruby

ruby中线程的执行会产生竞争条件(race condition)和死锁,因为多个线程之间会不断切换时间片分别执行,而不是同步执行,但是我们可以通过mutex互斥锁来实现线程内代码同步执行,但也正是如此,有可能也会造成死锁的情况,比如线程1内获取了锁1,线程2内获取了锁2,紧接着线程去获取锁2,这一步需要等待锁2被释放,而线程2内需要获取锁1之后再去释放锁2,需要线程1和线程2就开始互相等待释放锁,就死锁了,代码无法继续执行下去


  require 'thread'

  m,n = Mutex.new, Mutex.new
  
  t = Thread.new {
    m.lock
    puts "Thread t locked Mutex m"
    # sleep 1
    puts "Thread t waiting to lock Mutex n"
    n.lock
  }
  
  s = Thread.new {
    n.lock
    puts "Thread s locked Mutex n"
    # sleep 1
    puts "Thread s waiting to lock Mutex m"
    m.lock
  }
  
  t.join
  s.join

事实上只要有一个文件io操作或者任何其他io(网络io)操作,就会使得ruby线程开始切换时间片,让cpu来处理其他线程的代码:

def dl_test
  m1 = Mutex.new
  m2 = Mutex.new
  t1 = Thread.new do
    m1.lock
    m2.lock
    m2.unlock1
    m1.unlock
  end
  t2 = Thread.new do
    m2.lock
    m1.lock
    m1.unlock
    m2.unlock
  end
  t1.join
  t2.join
end

以上代码的线程中分别获取m1和m2的锁,上面的代码出现死锁的概率非常小,大多数情况下,执行顺序就是(以t1为例):获取m1的锁,获取m2的锁,释放m2的锁,释放m1的锁,然后t2开始执行,但是在获取另一个锁的时候,线程执行是有可能发生切换的,也就是t1执行的时候,有可能在获取m2的锁的时候发生时间片切换,只需要增加一个puts操作,就可以促使时间片切换:

def dl_test
  m1 = Mutex.new
  m2 = Mutex.new
  t1 = Thread.new do
    m1.lock
    puts "*"
    m2.lock
    m2.unlock
    m1.unlock
  end
  t2 = Thread.new do
    m2.lock
    m1.lock
    m1.unlock
    m2.unlock
  end
  t1.join
  t2.join
end

这样子就会发生死锁,t1获取m1锁,然后因为执行puts阻塞操作,cpu转而执行t2线程,t2获取m2锁,之后不管时间片是否继续切换都会发生死锁了,如果不切换,t2需要获取m1锁,但是m1锁还没释放,如果切换到t1,t1需要获取m2锁,但是m2锁也没有被t2释放,最终只会导致两个线程在互相等待锁释放,导致死锁.
