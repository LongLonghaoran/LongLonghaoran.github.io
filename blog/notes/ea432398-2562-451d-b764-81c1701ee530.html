1.js高级部分:
    1.三种创建对象的方式
        1.字面量
            var per = {
                name: '名字'
            }
        2.调用系统的构造函数
            var obj = new Object()
            obj.name = '名字'
            obj.sex = '男'
        3.自定义构造函数
            function Person(name, sex){
                this.name;
                this.sex;
                this.sayHi = function(){
                    console.log("你好啊");
                }
            }
            var person = new Person();

        构造函数内部原理，三段式:
            三段式的前提，要通过new生成对象
            1.一个函数，只要通过new操作，内部就会先隐式生成 var this = {__proto__: Object.prototype}，函数体尾部隐式加上return this,所以最后返回的是个对象
            2.调用this.name = xxx,赋值操作
            3.隐式的返回 this
    2.实例对象和构造函数之间的关系:
        console.dir 可以直接展示对象的结构

        实例对象是通过构造函数来创建的

        当我们定义了一个构造函数的时候，通过查看构造函数可以看到，构造函数
        在它的名称.prototype中可以发现构造函数Person
        function Person(){

        }
        或者可以这么说: Person.prototype.constructor = Person == per.constructor == per.__proto__.constructor，四者是相等的
        通过实例对象的constructor方法也可以获取实例对象的构造函数/构造器
        总结:
            1.实例对象是通过高走函数来创建的
            2.如何判断是不是这个数据类型?
                1.通过判断 实例对象.构造器==构造函数名字
                2.对象 instanceof 构造函数名字
                尽可能使使用第二种方式来识别
    3.原型的引入:
        1.通过构造函数定义属性和方法带来的问题:
            还是这个构造函数:
                functio Person(name, age){
                    this.name = name;
                    this.age =age;
                    this.sayHi = function(){

                    }
                }
            如果我们一次创建2个对象:
                var per1 =new Person();
                var per2 = new Person();
                per1.eat == per2.eat
            那么实际的效果将会是创建了100个不同的对象，并且这2个对象中
            的函数都是不同的、单独定义的,这样非常浪费内存,因为像
            sayHi这样的方法,每个对象都是一模一样，并不需要另外去定义,
            我们需要解决这个问题:
                1.我们可以将函数定义从构造函数中抽出来，只给它指定一个命名函数的名字
                    function Person(){
                        this.eat = f;
                    }
                    function f(){

                    }
                    这样做有它的缺点,很容易命名冲突了,而且函数定义没有和构造函数绑定在
                    一起,这样写法也不严谨
                2.通过原型来解决来解决数据共享的问题:
                    所以我们需要原型来定义所有对象共有的属性和方法
                    function Person(name){
                        this.name = name;
                    }
                    Person.prototype.eat = function(){};
        2.Person是构造函数 Person.prototype就是它的原型对象,使用构造函数
            实例化对象new Person之后,会将原型赋予给实例化对象的__proto__属性.

            实例对象中存放的是__proto__ 构造函数中存放的是prototype

            对象在调用属性/方法时除了在自身对象中查找，还会去原型中查找
        3.有可能碰到的一个小坑
            如果在构造函数给属性赋值元素对象，并且给它注册事件，那么事件
            函数中的this和构造函数之后实例化的对象中this不要混淆了:
                function ChangeStyle(btnObj){
                    //  先将this保存出来待会备用
                    var that = this;
                    this.btnObj = btnObj;
                    this.btnObj.onclick = function(){
                        //现在已经处于事件函数中了，这个时候this指代的是事件的触发对象btnObj了，而不是上级的实例化对象了
                        //这个时候可使用刚才备用的that
                    }
                }
        4.总结：
            1.实例对象有个__proto__属性，叫 原型，不是标准属性，给浏览器使用的
                为什么说它不标准，因为仅仅只是火狐和谷歌浏览器恰巧给它开发权限，让我们
                可以使用这个属性，其实其他大部分浏览器都是不支持，而且按照浏览器标准
                这个属性本来就不应该展现出来的。。
                一般情况我们讲的原型对象，讲的都是 prototype
            2.构造函数有个属性，prototype,也叫原型，程序员使用
    4.构造函数 实例对象 原型对象之间的关系
            构造函数: {
                prototype原型对象: {
                    constructor: 构造函数(就是它自己)
                }
            }
            通过构造函数来实例化对象: var 对象名 = new 构造函数();
            对象名.__proto__ = 构造函数.prototype
    5.顺带一提,通过字面量 {}创建的对象，它的原型就是 Object
    6.简单的原型写法
        Person.prototype = {
            name: "188",
            weight: '55kg',
            study: function(){
                console.log('学习好开心啊');
            }
        }
        但是上面创建的对象有一个问题，__proto__原型中少了个构造器constructor
        所以我们应该手动加上:
            Person.prototype = {
                constructor: Person,
                name: "188",
                weight: '55kg',
                study: function(){
                    console.log('学习好开心啊');
                }
            }
    7.原型中的方法和构造函数中的this，都是指代实例化后的对象
    8.原型中的方法也是相互访问的:
        我们知道构造函数中定义的实例方法，是可以相互调用的，加上this.即可
            function Person(){
                this.play(){
                    this.eat();
                };
                this.eat = function(){
                    console.log("eating");
                }
            }
        同样，原型中的方法也可以相互访问:
            function Person(){

            }
            Person.prototype.play =function(){
                this.eat();
            }
            Person.prototype.eat = function(){
                console.log('eating');
            }
    9.实例对象使用的属性或者方法，都是先在实例中查找，找到了则直接使用，
        找不到则去实例对象的原型中找
    10.可以给内置对象添加方法
        比如 Array.unique
        Array.prototype.unique = function(){
            var result = {};
            var arr = [];
            this.forEach(function(e){
                result[e] = '存在';
            });
            for(var e in result){
                arr.push(Number(e));
            }
            return arr;
        }
        [1,2,3,3,4,4].unique();
    11.将局部变量变为全局变量,有个时候（少数时候）我们会需要自定义一个
        构造函数，用于在需要的时候实例化它来使用,这个时候可以将其赋给window
        对象,那么之后只要直接调用构造函数名称既可直接使用
        (function(win){
            function Random(){};
            Random.prototype.getRandom = function(min, max){
                return Math.random()*(max - min)+min;
            }
            win.Random = Random;
        })(window)
