Package行为
默认行为是根据import paths(导入路径)展示包列表或者单个包，用以判断一个包是否存在，在go package主要判断一个包是否存在与GOPATH/src中
使用-f可以通过字符模板指定输出格式,模板中可以使用的方法即为一个包对象的struct的字段,默认是调用该包对象的String方法
type Package struct {
        Dir           string   // directory containing package sources
        ImportPath    string   // import path of package in dir
        ImportComment string   // path in import comment on package statement
        Name          string   // package name
        Doc           string   // package documentation string
        Target        string   // install path
        Shlib         string   // the shared library that contains this package (only set when -linkshared)
        ... ...
}
比如: go list -f '{{.Target}}'

vendor目录下的包/Packages反馈了包含到vendor目录的导入路径ImportPath，所以导入路径是可以唯一标识一个包的。
如果有错误信息的话，则是
    type PackageError struct {
        ImportStack   []string // shortest path from package named on command line to this one
        Pos           string   // position of error (if present, file:line:col)
        Err           string   // the error itself
    }

模板中还可以使用join和context,join表示strings.Join方法,context表示Context结构体
    type Context struct {
        GOARCH        string   // target architecture
        GOOS          string   // target operating system
        GOROOT        string   // Go root
        GOPATH        string   // Go path
        CgoEnabled    bool     // whether cgo can be used
        UseAllFiles   bool     // use files regardless of +build lines, file names
        Compiler      string   // compiler to assume when computing target paths
        BuildTags     []string // build constraints to match in +build lines
        ToolTags      []string // toolchain-specific build constraints
        ReleaseTags   []string // releases the current release is compatible with
        InstallSuffix string   // suffix to use in the name of the install dir
    }
比如 go list -f '{{context.GOARCH}}'

Module下行为
事实上，现在几乎所有golang项目都已经切换到gomodule模式
模块module信息与包package类似，也是有一个结构体type Module struct
[图片/附件]
-m选项使得go list命令可以列出module模块
-f选项，模块也有一个结构体type Module struct，配合-f选项可以只展示指定的字段
使用go list -m all以列出当前module使用的所有module, go list命令是根据go.mod文件来查询module信息的
-u选项会在输出中添加可用的更新的，-u会设置module的Update字段为更新的模块的信息。如果当前正在使用的版本，已经废弃了，那么还会设置module的Retracted字段来表示；module的String方法也会指示可用的更新，它会在输出中的最后用括号将新版本括起来。
[图片/附件]
-u选项会在输出中，在每个模块后追加展示该模块可用的所有可更新的版本
[图片/附件]
go list -m列出哪些模块?
1.主模块: 当前目录所在的模块就是主模块,而模块就是含有go.mod的目录
2.active module:当前主模块以及它的依赖模块
3.如果go list -m 后面没有带参数的话，那么默认就是仅列出主模块，或者指定一个模块名那么就列出该模块，这个模块名必须是当前模块所依赖的模块才可以；all参数表示将所有active module全部列出来；...三个点表示通配符；path@version这样的格式用于限定要查找的模块的版本。
